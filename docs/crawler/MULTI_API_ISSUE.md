# 多接口页面分割问题说明

## 问题描述

企业微信的某些 API 文档页面（如文档 99935）在一个页面中包含多个独立的 API 接口，但参数表格是混杂在一起的，导致自动分割时每个接口都会获取到所有接口的参数。

### 示例：文档 99935

该文档包含 5 个 API 接口：
1. 查询智能表格子表权限 (`get_sheet_priv`)
2. 更新智能表格子表权限 (`update_sheet_priv`)
3. 新增智能表格指定成员额外权限 (`create_rule`)
4. 更新智能表格指定成员额外权限 (`mod_rule_member`)
5. 删除智能表格指定成员额外权限 (`delete_rule`)

### 页面结构问题

```markdown
## 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| docid | string | 是 | 第1个接口的参数 |
| type | uint32 | 是 | 第1个接口的参数 |
| docid | string | 是 | 第2个接口的参数 |
| rule_id | uint32 | 是 | 第2个接口的参数 |
| docid | string | 是 | 第3个接口的参数 |
| name | string | 是 | 第3个接口的参数 |
...（所有接口的参数混在一起）

## 其他说明

### 查询智能表格子表权限
请求方式：POST（HTTPS）
请求地址: https://qyapi.weixin.qq.com/.../get_sheet_priv
（纯文本格式，没有表格结构）

### 更新智能表格子表权限
请求方式：POST（HTTPS）
请求地址: https://qyapi.weixin.qq.com/.../update_sheet_priv
（纯文本格式，没有表格结构）
```

**问题**：
1. 顶部的参数表格包含了所有接口的参数混在一起
2. "其他说明"部分每个接口的参数是纯文本格式，没有表格结构
3. 自动分割时无法准确区分哪些参数属于哪个接口

## 解决方案

### 方案 1：禁用自动分割（默认，推荐）

**使用场景**：大多数情况

**优点**：
- ✅ 保持原始页面完整性
- ✅ 不会丢失任何信息
- ✅ 参数表格完整可用

**缺点**：
- ❌ 一个文档包含多个接口
- ❌ 需要手动区分不同接口

**使用方法**：
```bash
# 默认行为，不需要额外参数
python3 crawler.py --doc-ids 99935
```

**输出**：
```
99935-6.md  # 包含所有 5 个接口的完整文档
```

### 方案 2：启用自动分割（实验性）

**使用场景**：想要尝试自动分割，接受参数混杂的风险

**优点**：
- ✅ 每个接口独立成文档
- ✅ 文件名包含 API 名称

**缺点**：
- ❌ 参数会重复和混杂
- ❌ 每个接口都包含所有接口的参数
- ❌ 可能导致混淆

**使用方法**：
```bash
# 启用分割（实验性功能）
python3 crawler.py --doc-ids 99935 --split-multi-api
```

**输出**：
```
99935-1-get_sheet_priv-6.md       # 包含所有接口的参数
99935-2-update_sheet_priv-6.md    # 包含所有接口的参数
99935-3-create_rule-5.md          # 包含所有接口的参数
99935-4-mod_rule_member-6.md      # 包含所有接口的参数
99935-5-delete_rule-6.md          # 包含所有接口的参数
```

### 方案 3：手动处理（最准确）

**使用场景**：需要精确的接口文档

**步骤**：
1. 使用默认模式爬取完整文档
2. 手动查看"其他说明"部分的每个接口详情
3. 根据需要手动创建独立的接口文档

## 建议

### 对于普通使用

**推荐**：使用默认模式（不启用 `--split-multi-api`）

```bash
python3 crawler.py --doc-ids 99935
```

这样可以获得完整的文档，所有参数都在一起，可以：
1. 查看完整的参数表格
2. 在"其他说明"部分查看每个接口的详细说明
3. 根据接口 URL 区分不同的接口

### 对于代码生成

如果你要基于这些文档生成代码，建议：

1. **不要使用自动分割**，而是：
   - 从"其他说明"部分提取每个接口的 API URL
   - 手动映射参数到对应的接口
   - 或者改进爬虫，解析"其他说明"部分的纯文本格式

2. **改进建议**：
   ```python
   # 可以在代码生成时，根据 API URL 来过滤参数
   api_url = "get_sheet_priv"
   
   # 从 JSON 请求示例中提取该接口的参数
   # 而不是从混杂的表格中提取
   ```

## 技术细节

### 为什么不能完美分割？

1. **参数表格是共享的**
   - 原始页面将所有接口的参数放在一个表格中
   - 没有明确的分隔标记

2. **"其他说明"部分是纯文本**
   - 参数说明没有表格结构
   - 格式不统一，难以解析

3. **参数重复**
   - 多个接口使用相同的参数名（如 `docid`）
   - 无法通过参数名区分
