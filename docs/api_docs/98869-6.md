# 设置高级布局 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98869](https://developer.work.weixin.qq.com/document/path/98869)
- **文档 ID**: `98869`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/advanced_layout/apply?access_token=ACCESS_TOKEN`

## 接口描述

将会议中的高级自定义布局应用到指定成员或者整个会议。也可以恢复指定成员或整个会议的默认布局。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 是 | 会议 ID |
| layout_id | string | 是 | 选择应用的布局 ID（若传空""，表示恢复成当前会议的默认布局）。应用布局的优先级从高到低为：个性布局 > 自定义布局 > 默认布局（MRA 不支持同框模式， 如果会议设置为同框模式，MRA 应用默认布局。） |
| user_list | object[] | 否 | 用户列表对象数组。如果该字段为空，为会议设置高级自定义布局；如果该字段携带用户，则只为指定用户设置个性布局。单次最多支持20个用户。详见User |
| tmp_openid | string | 是 | 用户当前会议中的临时身份 ID，单场会议唯一。仅对H.323/SIP 会议室终端生效 |

### 请求示例

```json
{"meetingid":"MEETINGID","layout_id":"LAYOUTID1","user_list":[{"tmp_openid":"TMPOPENID1"}]}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
