# 获取部门列表

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/90344](https://developer.work.weixin.qq.com/document/path/90344)
- **文档 ID**: `90344`
- **请求方法**: `GET`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/department/list`

## 接口描述

企业通讯录安全特别重要，企业微信将持续升级加固通讯录接口的安全机制。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |
| id | string | 否 | 部门id。获取指定部门及其下的子部门（以及子部门的子部门等等，递归）。 如果不填，默认获取全量组织架构 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| department | object[] | 部门列表数据 |
| department[].id | int32 | 创建的部门id |
| department[].name | string | 部门名称，代开发自建应用需要管理员授权才返回；此字段从2019年12月30日起，对新创建第三方应用不再返回，2020年6月30日起，对所有历史第三方应用不再返回name，返回的name字段使用id代替，后续第三方仅通讯录应用可获取，未返回名称的情况需要通过通讯录展示组件来展示部门名称 |
| department[].name_en | string | 英文名称，此字段从2019年12月30日起，对新创建第三方应用不再返回，2020年6月30日起，对所有历史第三方应用不再返回该字段 |
| department[].department_leader | string[] | 部门负责人的UserID；第三方仅通讯录应用可获取 |
| department[].parentid | int32 | 父部门id。根部门为1 |
| department[].order | int32 | 在父部门中的次序值。order值大的排序靠前。值范围是[0, 2^32) |

## 其他说明

### 权限说明

只能拉取token对应的应用的权限范围内的部门列表
