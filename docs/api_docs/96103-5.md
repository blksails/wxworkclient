# 设备数据授权变更回调

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/96103](https://developer.work.weixin.qq.com/document/path/96103)
- **文档 ID**: `96103`
- **请求方法**: `POST`
- **接口地址**: `https://127.0.0.1/suite/receive`

## 接口描述

当用户授权设备数据给应用发生变更时，企业微信回调该指令回调到应用配置的数据回调url中。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| msg_signature | string | 是 | 消息签名 |
| timestamp | string | 是 | 时间戳 |
| nonce | string | 是 | 随机数 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| SuiteId | string | 是 | 第三方应用的SuiteId |
| AuthCorpId | string | 是 | 授权企业的CorpID |
| InfoType | string | 是 | 事件类型，device_data_auth_change 设备数据授权变更 |
| TimeStamp | int32 | 是 | 消息创建时间（整型） |
| AgentID | int32 | 是 | 企业应用的id，整型。可在应用的设置页面查看 |

### 请求示例

```xml
<xml>
  <SuiteId><![CDATA[ww4asffe99e54cxxxx]]></SuiteId>
	<AuthCorpId><![CDATA[wxf8b4f85f3a79xxxx]]></AuthCorpId>
	<InfoType><![CDATA[device_data_auth_change]]></InfoType>
	<TimeStamp>1403610513</TimeStamp>
	<AgentID>1000001</AgentID>
</xml>
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误信息 |
