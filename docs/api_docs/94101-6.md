# 查询客户接替状态

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94101](https://developer.work.weixin.qq.com/document/path/94101)
- **文档 ID**: `94101`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/externalcontact/resigned/transfer_result`

## 接口描述

企业和第三方可通过此接口查询离职成员的客户分配情况。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| handover_userid | string | 是 | 原添加成员的`userid` |
| takeover_userid | string | 是 | 接替成员的`userid` |
| cursor | string | 否 | 分页查询的cursor，每个分页返回的数据不会超过1000条；不填或为空表示获取第一个分页 |

### 请求示例

```json
{
   "handover_userid": "zhangsan",
   "takeover_userid": "lisi",
   "cursor":"CURSOR"
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| customer | object[] | 转接客户信息列表 |
| customer[].external_userid | string | 转接客户的外部联系人userid |
| customer[].status | int32 | 接替状态， 1-接替完毕 2-等待接替 3-客户拒绝 4-接替成员客户达到上限 |
| customer[].takeover_time | int32 | 接替客户的时间，如果是等待接替状态，则为未来的自动接替时间 |
| next_cursor | string | 下个分页的起始cursor |

### 响应示例

```json
{
   "errcode": 0,
   "errmsg": "ok",
   "customer":
  [
  {
  	"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACCCC",
  	"status":1,
  	"takeover_time":1588262400
  },
  {
  	"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACBBBB",
  	"status":2,
  	"takeover_time":1588482400
  },
  {
  	"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACAAAA",
  	"status":3,
  	"takeover_time":0
  }
  ],
  "next_cursor":"NEXT_CURSOR"
}
```

## 其他说明

### 权限说明

企业需要使用“客户联系”secret或配置到“可调用应用”列表中的自建应用secret所获取的accesstoken来调用。第三方应用需拥有“企业客户权限->客户联系->在职继承”权限。接替成员必须在此第三方应用或自建应用的可见范围内。
