# 查询电子发票 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99451](https://developer.work.weixin.qq.com/document/path/99451)
- **文档 ID**: `99451`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/card/invoice/reimburse/getinvoiceinfo?access_token=ACCESS_TOKEN`

## 接口描述

接口说明：报销方在获得用户选择的电子发票标识参数后，可以通过该接口查询电子发票的结构化信息，并获取发票PDF文件。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| card_id |  | 是 | 发票id |
| encrypt_code |  | 是 | 加密code |
| card_id |  | 否 | 发票id |
| begin_time |  | 否 | 发票的有效期起始时间 |
| end_time |  | 否 | 发票的有效期截止时间 |
| fee |  | 否 | 发票加税合计金额，以分为单位 |
| title |  | 否 | 发票的抬头 |
| billing_time |  | 否 | 开票时间，为十位时间戳 |
| billing_no |  | 否 | 发票代码 |
| billing_code |  | 否 | 发票号码 |
| tax |  | 否 | 税额,以分为单位 |
| fee_without_tax |  | 否 | 不含税金额，以分为单位 |
| name |  | 否 | 项目（商品）名称 |
| num |  | 否 | 项目数量 |
| unit |  | 否 | 项目单位 |
| price |  | 否 | 单价，以分为单位 |

### 请求示例

```json
{"card_id":"CARDID","encrypt_code":"ENCRYPTCODE"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| openid |  | 用户标识 |
| type |  | 发票类型，如广东增值税普通发票 |
| payee |  | 发票的收款方 |
| detail |  | 发票详情 |
| user_info |  | 发票的用户信息，见user_info结构说明 |
| detail |  | 发票详情，一般描述的是发票的使用说明 |
| pdf_url |  | 这张发票对应的PDF_URL |
| trip_pdf_url |  | 其它消费凭证附件对应的URL，如行程单、水单等 |
| check_code |  | 校验码 |
| buyer_number |  | 购买方纳税人识别号 |
| buyer_address_and_phone |  | 购买方地址、电话 |
| buyer_bank_account |  | 购买方开户行及账号 |
| seller_number |  | 销售方纳税人识别号 |
| seller_address_and_phone |  | 销售方地址、电话 |
| seller_bank_account |  | 销售方开户行及账号 |
| remarks |  | 备注 |
| cashier |  | 收款人，发票左下角处 |
| maker |  | 开票人，发票右下角处 |
| reimburse_status |  | 发报销状态INVOICE_REIMBURSE_INIT：发票初始状态，未锁定；INVOICE_REIMBURSE_LOCK：发票已锁定；INVOICE_REIMBURSE_CLOSURE：发票已核销 |
| info |  | 商品信息结构，见下方说明 |
