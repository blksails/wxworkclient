# 获取群聊列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/100989](https://developer.work.weixin.qq.com/document/path/100989)
- **文档 ID**: `100989`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartsheet/groupchat/list?access_token=ACCESS_TOKEN`

## 接口描述

该接口可以查询通过智能表格自动化创建的群聊列表。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| docid |  | 是 | 智能表格ID。需为当前应用创建的智能表格 |
| cursor |  | 否 | 用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |
| limit |  | 否 | 用于分页查询，每次请求返回的数据上限。默认100，最大200注意：不保证每次返回的数据刚好为指定limit，必须用返回的has_more判断是否继续请求 |

### 请求示例

```json
{"docid":"DOCID","cursor":"CURSOR","limit":2}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| has_more |  | 是否还有更多数据未获取 |
| next_cursor |  | 下一次请求的cursor值 |
| chat_id_list |  | 符合条件的群聊chatid列表 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","has_more":true,"next_cursor":"GNIJIGEO","chat_id_list":["chatid1","chatid2"]}
```
