# 获取考勤打卡原始数据

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/96027](https://developer.work.weixin.qq.com/document/path/96027)
- **文档 ID**: `96027`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/devicedata/get_checkin_data`

## 接口描述

第三方应用获取企业已授权的考勤机打卡记录。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 应用调用接口权限 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_type | uint32 | 是 | 成员类型，0-企业成员，2-学生 |
| begin_time | uint32 | 是 | 表示设备上传记录的开始时间戳 |
| end_time | uint32 | 是 | 表示设备上传记录的结束时间戳，时间跨度不超过一个月 |
| data_filter_type | uint32 | 是 | 打卡数据筛选条件，1-按设备sn进行筛选，2-按成员openuserid进行筛选 |
| device_sn_list | string | 否 | 打卡设备SN列表，最多传入100个 |
| open_userid_list | string | 否 | 成员openuserid列表，最多传入100个，须在应用可见范围 |
| cursor | string | 否 | 分页查询游标，上一次调用时返回的next_cursor，初次调用可以不填。调用方需保证cursor的正确性 |
| limit | uint32 | 否 | 分页查询限制，最多可返回limit条数据，不填默认200，最多500 |
| agentid | uint32 | 否 | 应用agentid，第三方套件应用必填，第三方单应用可不填 |

### 请求示例

```json
{
	"user_type": 0,
	"begin_time":12345,
	"end_time":67890,
	"data_filter_type": 1,
	"device_sn_list":["SN1", "SN2"],
	"cursor":"CURSOR",
	"limit":"LIMIT",
	"agentid":10000
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码描述 |
| checkindata | object | 打卡数据 |
| checkindata.items | object[] | 打卡数据列表 |
| checkindata.items.open_userid | string | 打卡人员open_userid |
| checkindata.items.checkin_time | uint32 | 打卡时间戳 |
| checkindata.items.device_sn | string | 打卡设备SN |
| next_cursor | string | 分页游标，用于实现分页拉取 |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"checkindata": {
		"items":[
			{
				"open_userid": "x1",
				"checkin_time": 12345,
				"device_sn": "SN"
			}
		]
	},
	"next_cursor":"NEXT_CURSOR"
}
```

## 其他说明

### 注意

1.第三方应用须具有“设备接口信息权限-考勤打卡原始数据”授权；
2.企业须同时安装硬件设备和第三方应用；
3.企业须向第三方应用二次授权具体硬件设备的原始数据；
4.仅返回应用可见范围内的成员打卡数据；
5.接口返回数据按照打卡记录上传时间排序；
6.请求时间跨度应在一个月内，并且跨月请求易返回重复数据，请尽量避免；
7.部分场景下，接口可能返回重复的打卡数据，调用端应进行排重。
