# 创建预约直播

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93717](https://developer.work.weixin.qq.com/document/path/93717)
- **文档 ID**: `93717`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/living/create`

## 接口描述

创建预约直播

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| anchor_userid | string | 是 | 直播发起者的userid |
| theme | string | 是 | 直播的标题，最多支持20个utf8字符 |
| living_start | uint32 | 是 | 直播开始时间的unix时间戳 |
| living_duration | uint32 | 是 | 直播持续时长 |
| type | uint32 | 否 | 直播的类型，0：通用直播，1：小班课，2：大班课，3：企业培训，4：活动直播，默认 0。其中大班课和小班课仅k12学校和IT行业类型能够发起 |
| description | string | 否 | 直播的简介，最多支持100个utf8字符，仅对“通用直播”、“小班课”、“大班课”和“企业培训”生效，“活动直播”简介通过activity_detail.description控制 |
| agentid | uint32 | 否 | 授权方安装的应用agentid。仅旧的第三方多应用套件需要填此参数 |
| remind_time | uint32 | 否 | 指定直播开始前多久提醒用户，相对于living_start前的秒数，默认为0 |
| activity_cover_mediaid | string | 否 | 活动直播特定参数，直播间封面图的mediaId |
| activity_share_mediaid | string | 否 | 活动直播特定参数，直播分享卡片图的mediaId |
| activity_detail.description | string | 否 | 活动直播特定参数，活动直播简介 |
| activity_detail.image_list | string[] | 否 | 活动直播特定参数，活动直播附图的mediaId列表，最多支持传5张，超过五张取前五张 |

### 请求示例

```json
{
   "anchor_userid": "zhangsan",
   "theme": "theme",
   "living_start": 1600000000,
   "living_duration": 3600,
   "description": "test description",
   "type": 4,
   "agentid" : 1000014,
   "remind_time": 60,
   "activity_cover_mediaid": "MEDIA_ID",
   "activity_share_mediaid": "MEDIA_ID",
   "activity_detail":
   {
       "description": "活动描述，非活动类型的直播不用传",
       "image_list": [
            "MEDIA_ID_1",
            "MEDIA_ID_2"
       ]
   }
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| livingid | string | 直播id，通过此id可调用“进入直播”接口(包括小程序接口和JS-SDK接口)，以实现主播到点后的开播操作，以及观众进入直播详情预约和观看直播 |

### 响应示例

```json
{
   "errcode": 0,
   "errmsg": "ok",
   "livingid": "XXXXXXXXX"
}
```
