# 获取控制器列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98799](https://developer.work.weixin.qq.com/document/path/98799)
- **文档 ID**: `98799`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/rooms/list_controllers?access_token=ACCESS_TOKEN`

## 接口描述

获取企业下的控制器列表。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| controller_name | string | 否 | 需要获取的控制器名称（支持模糊匹配查找） |
| cursor | string | 否 | 分页查询用，将上一个请求返回的next_cursor字段传入。第一次查询时可不传值 |
| limit | uint32 | 否 | 分页大小，从1开始，最大50，默认20 |

### 请求示例

```json
{"controller_name":"NAME","cursor":"CURSOR","limit":20}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| has_more | bool | 是否还有更多Rooms会议室列表 |
| next_cursor | string | 分页用，has_more为true时，下一次拉取列表将该字段填入cursor字段 |
| controller_info_list | object[] | Rooms会议室对象列表，详见ControllerInfo |
| rooms_id |  | string |
| meeting_room_name |  | string |
| meeting_room_location |  | string |
| controller_name |  | string |
| manufacture_name |  | string |
| controller_model |  | string |
| app_version |  | string |
| status |  | string |
| framework_version |  | string |
| ip_address |  | string |
| mac_address |  | string |
| cpu_type |  | string |
| cpu_usage |  | string |
| network_type |  | string |
| mem_usage |  | string |
