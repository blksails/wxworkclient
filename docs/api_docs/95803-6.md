# 为打卡人员补卡 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95803](https://developer.work.weixin.qq.com/document/path/95803)
- **文档 ID**: `95803`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/checkin/punch_correction?access_token=ACCESS_TOKEN`

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |
| userid | string | 是 | 需要补卡的成员userid |
| schedule_date_time | uint32 | 是 | 应打卡日期，为当天0点的Unix时间戳。 |
| schedule_checkin_time | uint32 | 否 | 应打卡时间点，相对应打卡日期0点的偏移秒数，如9点整则为32400。可通过获取员工打卡规则获取对应的规则打卡时间点，如work_sec/off_work_sec。对于没有规则对应的打卡时间点，如休息日打卡、无规则打卡、自由上下班，该参数不用填。 |
| checkin_time | uint32 | 是 | 实际打卡时间，Unix时间戳。相对于schedule_checkin_time的实际打卡时间，具体可以表现为正常/迟到/早退 |
| remark | string | 否 | 备注信息不超过512字节 |
| 自建应用 | 自建应用 | 否 | 配置到「打卡 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"userid":"zhangsan","schedule_date_time":1654444800,"schedule_checkin_time":32400,"checkin_time":1654486827,"remark":"备注信息"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 错误码描述 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
