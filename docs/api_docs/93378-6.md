# 录入打卡人员人脸信息 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93378](https://developer.work.weixin.qq.com/document/path/93378)
- **文档 ID**: `93378`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/checkin/addcheckinuserface?access_token=ACCESS_TOKEN`

## 接口描述

企业可通过自建应用，为企业打卡人员录入人脸信息，人脸信息仅用于人脸打卡。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证，获取方式参考：文档-获取access_token |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userid | string | 否 | 需要录入的用户id |
| userface | string | 否 | 需要录入的人脸图片数据，需要将图片数据base64处理后填入，对已录入的人脸会进行更新处理 |
| 自建应用 | 自建应用 | 否 | 配置到「打卡 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"userid":"james","userface":"PLACE_HOLDER"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 错误码描述 |
| 301021 |  | 输入参数错误 |
| 301069 |  | 输入userid无对应成员 |
| 301070 |  | 系统错误，请稍后再试 |
| 301071 |  | 企业内有其他人员有相似人脸，此情况下人脸仍然会录入成功 |
| 301072 |  | 人脸图像数据错误请更换图片 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
