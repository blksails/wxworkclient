# 获取会议的外呼状态 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98824](https://developer.work.weixin.qq.com/document/path/98824)
- **文档 ID**: `98824`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/phone/get_callout_status?access_token=ACCESS_TOKEN`

## 接口描述

此接口用于获取外呼电话入会的数据。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 是 | 会议ID |
| cursor | string | 否 | 分页查询用，将上一个请求返回的next_cursor字段传入。第一次查询时可不传值。 |
| limit | uint32 | 否 | 分页大小，默认20，最大100。 |

### 请求示例

```json
{"meetingid":"MEETINGID","cursor":"CURSOR","limit":10}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| phone_numbers | object[] | 电话号码对象数组。详见PhoneNumber。 |
| next_cursor | string | 分页用，下一次拉取列表将该字段填入cursor字段 |
| has_more | bool | 是否还有待拉取的列表 |
| area | uint32 | 国家/地区代码。（例如：中国是86） |
| phone | string | 电话号码或固定电话总机号 |
| extension_number | string | 固定电话分机号 |
| status | string | 外呼状态 |
| tmp_openid | string | 当场会议的成员临时 ID，可用于会控操作，适用于所有成员。 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","phone_numbers":[{"area":86,"phone":"13333333333","extension_number":"12345678","status":"STATUS","tmp_openid":"TMP_OPENID"}],"next_cursor":"NEXT_CURSOR","has_more":false}
```
