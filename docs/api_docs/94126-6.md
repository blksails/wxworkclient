# 获取设备打卡数据 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94126](https://developer.work.weixin.qq.com/document/path/94126)
- **文档 ID**: `94126`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/hardware/get_hardware_checkin_data?access_token=ACCESS_TOKEN`

## 接口描述

应用可通过此接口，获取可见范围内成员在考勤设备上产生的原始打卡记录，包括未被打卡应用记录的不符合打卡规则的记录。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证。 |
| filter_type |  | 否 | 过滤类型，1表示按打卡时间过滤，2表示按设备上传打卡记录的时间过滤，默认值是1 |
| starttime |  | 是 | Unix时间戳，当filter_type为1时，表示打卡的开始时间；当filter_type为2时，表示设备上传记录的开始时间 |
| endtime |  | 是 | Unix时间戳，当filter_type为1时，表示打卡的结束时间；当filter_type为2时，表示设备上传记录的结束时间 |
| useridlist |  | 是 | 需要获取打卡记录的用户列表 |
| 自建应用 | 自建应用 | 否 | 配置到「打卡 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 具有「打卡」权限, 且设备型号需关联该第三方应用 |

### 请求示例

```json
{"filter_type":1,"starttime":1492617600,"endtime":1492790400,"useridlist":["james","paul"]}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| userid |  | 用户id |
| checkin_time |  | 打卡时间。Unix时间戳 |
| device_sn |  | 打卡设备的sn |
| device_name |  | 打卡设备名 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","checkindata":[{"userid":"james","checkin_time":1492617610,"device_sn":"xxxxx","device_name":"xxxx门店"},{"userid":"paul","checkin_time":1492617620,"device_sn":"yyyy","device_name":"yyyy门店"}]}
```
