# 标记开票状态

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99437](https://developer.work.weixin.qq.com/document/path/99437)
- **文档 ID**: `99437`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/paytool/mark_invoice_status?provider_access_token=PROVIDER_ACCESS_TOKEN`

## 接口描述

服务商可以使用该接口标记某个应用订单发票的开票状态

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| provider_access_token | string | 是 | 服务商调用接口凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| order_id | string | 是 | 要标记开票状态的订单号 |
| oper_userid | string | 是 | 标记开票状态的操作人，操作人需要有「收银台-发票管理」的权限 |
| invoice_status | uint32 | 是 | 要标记的开票状态。 1-已开具纸质发票，并邮寄给客户 2-已开具电子发票，并发送至客户邮箱 3-取消开具发票，取消后企业可再次申请 若订单对应开票状态为已开票，此次标记将不生效 |
| invoice_note | string | 是 | 填写开票备注，例如发票邮寄单号等，客户侧可见。不超过200字节 |

### 请求示例

```json
{
	"order_id": "ORDERID",
	"oper_userid": "USERID",
	"invoice_status": 1,
	"invoice_note": "NOTE"
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 错误码描述 |

### 响应示例

```json
{
    "errcode": 0,
    "errmsg": "ok"
}
```
