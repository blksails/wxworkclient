# 手机号获取userid

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/91693](https://developer.work.weixin.qq.com/document/path/91693)
- **文档 ID**: `91693`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/user/getuserid`

## 接口描述

通过手机号获取其所对应的userid。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证，授权企业的token或上游获取的下游企业的token |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| mobile | string | 是 | 用户在企业微信通讯录中的手机号码。长度为5~32个字节 |

### 请求示例

```json
{
   "mobile": "13430388888"
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| userid | string | 成员UserID。对应管理端的账号，企业内必须唯一。不区分大小写，长度为1~64个字节。注意：第三方应用获取的值是密文的userid |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"userid": "zhangsan"
}
```

## 其他说明

### 权限说明

应用须拥有指定成员的查看权限。

### 更多说明

请确保手机号的正确性，若出错的次数超出企业人数上限的20%，会导致1天不可调用。
