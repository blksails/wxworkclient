# 获取高级功能账号列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99518](https://developer.work.weixin.qq.com/document/path/99518)
- **文档 ID**: `99518`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/wedoc/vip/list?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于查询企业已分配高级功能且在应用可见范围的账号列表。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| cursor |  | 否 | 用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |
| limit |  | 否 | 用于分页查询，每次请求返回的数据上限。默认100，最大200注意：不保证每次返回的数据刚好为指定limit，必须用返回的has_more判断是否继续请求 |
| 自建应用 | 自建应用 | 否 | 配置到「协作 - 文档 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"cursor":"CURSOR","limit":2}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| has_more |  | 是否还有更多数据未获取 |
| next_cursor |  | 下一次请求的cursor值 |
| userid_list |  | 符合条件的企业成员userid列表 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","has_more":true,"next_cursor":"GNIJIGEO","userid_list":["zhangsan","lisi"]}
```
