# 获取会议受邀成员列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98160](https://developer.work.weixin.qq.com/document/path/98160)
- **文档 ID**: `98160`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/get_invitees?access_token=ACCESS_TOKEN`

## 接口描述

预约会议时添加到参会人名单的成员，即为会议受邀成员。根据会议 ID 可获取受邀成员列表，支持分页获取

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 是 | 会议id |
| cursor | string | 否 | 分页查询用，将上一个请求返回的next_cursor字段传入。第一次查询时可不传值 |

### 请求示例

```json
{"meetingid":"meetingid11","cursor":"NIGEIJG"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| has_more | bool | 是否还有尚未拉取的成员列表 |
| next_cursor | string | 当“has_more”为 true 时，下次查询时输入参数“cursor”的值。 |
| invitees | object[] | 受邀成员列表，详见Invitee |
| userid | string | 成员userid |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","has_more":true,"next_cursor":"GENIG","invitees":[{"userid":"USERID"}]}
```
