# 切换 MRA 默认布局 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98787](https://developer.work.weixin.qq.com/document/path/98787)
- **文档 ID**: `98787`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/mra/set_default_layout?access_token=ACCESS_TOKEN`

## 接口描述

会议中对 MRA 的默认布局进行设置。如果当前 MRA 已显示会议自定义布局或个性布局或焦点视频，则不支持进行默认布局设置。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 是 | 会议ID |
| default_layout | int32 | 是 | 当前成员的默认分屏设置：1：等分模式2：全屏模式3：1+N |
| default_novideo_user | int32 | 是 | 默认非视频与会者在分屏中显示方式：1：显示2：隐藏 |

### 请求示例

```json
{"meetingid":"MEETINGID","default_layout":2,"default_novideo_user":1,"mra":{"tmp_openid":"TMP_OPENID"}}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| mra.tmp_openid | string | 被操作 mra 设备 的会中临时ID |
| errcode |  | int32 |
| errmsg |  | string |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
