# 获取门禁通行原始数据

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/96029](https://developer.work.weixin.qq.com/document/path/96029)
- **文档 ID**: `96029`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/devicedata/get_accesscontrol_data`

## 接口描述

第三方应用获取企业已授权的门禁设备通行记录。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 应用调用接口权限 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_type | uint32 | 是 | 成员类型，0-企业成员，2-学生 |
| begin_time | uint32 | 是 | 表示设备上传记录的开始时间戳 |
| end_time | uint32 | 是 | 表示设备上传记录的结束时间戳，时间跨度不超过一个月 |
| data_filter_type | uint32 | 是 | 门禁通行数据筛选条件，1-按设备sn进行筛选，该条件下device_sn_list必填；2-按成员openuserid进行筛选，该条件下open_userid_list必填 |
| device_sn_list | string | 否 | 门禁设备SN列表，最多传入100个 |
| open_userid_list | string | 否 | 成员openuserid列表，最多传入100个，须在应用可见范围，否则忽略 |
| cursor | string | 否 | 分页查询游标，上一次调用时返回的next_cursor，初次调用可以不填。调用方需保证cursor的正确性 |
| limit | uint32 | 否 | 分页查询限制，最多可返回limit条数据，不填默认200，最多500 |
| agentid | uint32 | 否 | 应用agentid，第三方套件应用必填，第三方单应用可不填 |

### 请求示例

```json
{
    "user_type":0,
    "begin_time":12345,
    "end_time":67890,
    "data_filter_type":1,
    "device_sn_list":[
        "SN1",
        "SN2"
    ],
    "open_userid_list":[

    ],
    "cursor":"CURSOR",
    "limit":"LIMIT",
    "agentid":10000
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码描述 |
| accesscontrol_data | object | 门禁通行数据 |
| accesscontrol_data.items | object[] | 门禁通行数据列表 |
| accesscontrol_data.items.open_userid | string | 通过门禁人员的open_userid |
| accesscontrol_data.items.user_type | uint32 | 人员类型：0-企业成员；1-访客；2-学生 |
| accesscontrol_data.items.timestamp | uint32 | 通过门禁的时间戳 |
| accesscontrol_data.items.pass_type | uint32 | 通行类型：1-进门；2-出门；3-无法识别方向 |
| accesscontrol_data.items.pass_method | uint32 | 通行方法：1-人脸；2-二维码；3-远程开门；4-指纹 |
| accesscontrol_data.items.device_sn | string | 测温设备SN |
| next_cursor | string | 分页游标，用于实现分页拉取，拉取下一页数据时需将前一页next_cursor填入请求cursor。若有返回该字段且为非空，表示可能还有满足的门禁数据，需继续调用接口；返回空字符串代表已经是最后一页。  由于权限检查等筛选，极端情况下可能accesscontrol_data.items返回为空，但后续依旧有满足条件的数据。 |
