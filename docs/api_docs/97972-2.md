# 微盘容量不足事件 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97972](https://developer.work.weixin.qq.com/document/path/97972)
- **文档 ID**: `97972`

## 接口描述

当企业微盘容量使用率超过90%，且企业有授权安装了具有微盘权限的第三方应用，则企业微信向第三方应用回调通知该事件。请开发者注意，该回调不是实时回调，而是每天定时检测触发回调，且单个授权企业每天最多回调一次。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ToUserName |  | 否 | 企业微信CorpID |
| FromUserName |  | 否 | 此事件该值固定为sys，表示该消息由系统生成 |
| CreateTime |  | 否 | 消息创建时间，unix时间戳 |
| MsgType |  | 否 | 消息类型，此时固定为：event |
| Event |  | 否 | 事件类型，此时固定为：wedrive_insufficient_capacity |

### 请求示例

```xml
<xml><ToUserName><![CDATA[toUser]]></ToUserName><FromUserName><![CDATA[sys]]></FromUserName><CreateTime>1348831860</CreateTime><MsgType><![CDATA[event]]></MsgType><Event><![CDATA[wedrive_insufficient_capacity]]></Event></xml>
```
