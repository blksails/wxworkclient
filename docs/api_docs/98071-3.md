# 扫描推广二维码事件 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98071](https://developer.work.weixin.qq.com/document/path/98071)
- **文档 ID**: `98071`
- **请求方法**: `POST`

## 接口描述

与本服务商已经有第三方应用授权关系的企业，其成员扫描推广二维码（包括移动端访问推广包注册链接）时，触发该事件，回调到服务商通用开发参数设置的系统事件接收URL上。请求方式：POST请求地址：https://127.0.0.1/service/receive?msg_signature=3a7b08bb8e6dbce3c9671d6fdb69d15066227608&timestamp=1403610513&nonce=380320359请求包体：xml请求示例：

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ServiceCorpId |  | 否 | 服务商corpid |
| InfoType |  | 否 | 事件类型，此处固定为enter_register_package |
| TimeStamp |  | 否 | 回调事件的时间戳 |
| AuthCorpId |  | 否 | 扫描推广二维码的成员所属企业corpid |
| AuthUserInfo |  | 否 | 扫描推广二维码的成员信息 |
| UserId |  | 否 | 成员的userid |
| RegisterCode |  | 否 | 注册链接对应的注册码 |
| State |  | 否 | 自定义的状态值，参数值由接口获取注册码指定。若未指定，则无该字段 |
| TemplateId |  | 否 | 推广包ID |
