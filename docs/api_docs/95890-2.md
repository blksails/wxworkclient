# 不同应用获取的ID不同

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95890](https://developer.work.weixin.qq.com/document/path/95890)
- **文档 ID**: `95890`
- **请求方法**: `GET`

## 接口描述

介绍了企业微信中基础的ID概念，包括corpid、userid、external_userid在不同应用类型下的取值。

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| 应用类型 | string | 应用的类型，包括自建应用、未升级的代开发应用与第三方应用、升级后的代开发应用与第三方应用 |
| corpid | string | 企业ID，根据应用类型不同可能是明文corpid或密文corpid |
| userid | string | 用户ID，根据应用类型不同可能是明文userid或密文userid |
| external_userid | string | 外部用户ID，企业主体下的external_userid或服务商主体下的external_userid |

## 其他说明

### 明文corpid

企业管理员可在管理端“我的企业”－“企业信息”下直接查看的“企业ID”。

### 明文userid

企业管理员可在管理端“通讯录”->点进某个成员的详情页中直接查看的“账号”。

### 企业主体下的external_userid

external_userid是企业的外部联系人ID，由企业微信系统生成，企业无法在管理端页面中查看，只能通过 API 获取到。

### 服务商主体下的密文corpid

服务商通过获取企业永久授权码或获取企业授权信息中所得到的corpid，与企业的明文corpid不同。

### 服务商主体下的密文userid

服务商通过获取部门成员等接口所得到的userid，不是企业的明文userid，而是加密之后的userid。

### 服务商主体下的external_userid

服务商通过客户联系相关接口中所获取到的external_userid，与企业主体下的external_userid不同。
