# 设置专区接收回调事件 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99963](https://developer.work.weixin.qq.com/document/path/99963)
- **文档 ID**: `99963`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/chatdata/set_receive_callback?access_token=ACCESS_TOKEN`

## 接口描述

应用可通过该接口设置专区中接收回调的程序id。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 自建应用 | 自建应用 | 否 | 需具备「数据与智能专区权限」 |
| 代开发应用 | 代开发应用 | 否 | 需具备「数据与智能专区权限」 |
| 第三方应用 | 第三方应用 | 否 | 需具备「数据与智能专区权限」 |
| program_id |  | 是 | 应用关联的程序id，同一个应用只能设置一个程序接收。若先设置了程序A接收，再调用该接口设置程序B时，会更改为程序B接收 |

### 请求示例

```json
{"program_id":"xxx"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 错误码 |
| errmsg |  | 错误码说明 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
