# 批量更新发票状态

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/90422](https://developer.work.weixin.qq.com/document/path/90422)
- **文档 ID**: `90422`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/card/invoice/reimburse/updatestatusbatch`

## 接口描述

发票平台可以通过该接口对某个成员的一批发票进行锁定、解锁和报销操作。注意，报销状态为不可逆状态，请开发者慎重调用。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| openid | string | 是 | 用户openid，可用["userid与openid互换接口"](#11279)获取 |
| reimburse_status | string | 是 | 发票报销状态 INVOICE_REIMBURSE_INIT：发票初始状态，未锁定；INVOICE_REIMBURSE_LOCK：发票已锁定，无法重复提交报销;INVOICE_REIMBURSE_CLOSURE:发票已核销，从用户卡包中移除 |
| invoice_list | object[] | 是 | 发票列表，必须全部属于同一个openid |
| invoice_list[].card_id | string | 是 | 发票卡券的card_id |
| invoice_list[].encrypt_code | string | 是 | 发票卡券的加密code，和card_id共同构成一张发票卡券的唯一标识 |

### 请求示例

```json
{
   "openid":"OPENID" ,
   "reimburse_status":"INVOICE_REIMBURSE_INIT",
    "invoice_list":
    [
        {"card_id":"cardid_1","encrypt_code":"encrypt_code_1"},
        {"card_id":"cardid_2","encrypt_code":"encrypt_code_2"}
    ]
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误信息 |

### 响应示例

```json
{
   "errcode":0,
   "errmsg":"ok",
}
```

## 其他说明

### 权限说明

仅认证的企业微信账号有接口权限

### 注

1. 报销方须保证在报销、锁定、解锁后及时将状态同步至微信端，保证用户发票可以正常使用
2. 批量更新发票状态接口为事务性操作，如果其中一张发票更新失败，列表中的其它发票状态更新也会无法执行，恢复到接口调用前的状态
