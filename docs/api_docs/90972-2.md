## 接口描述

该事件会回调给通讯录同步助手以及代开发自建应用

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ToUserName |  | 否 | 企业微信CorpID |
| FromUserName |  | 否 | 此事件该值固定为sys，表示该消息由系统生成 |
| CreateTime |  | 否 | 消息创建时间 （整型） |
| MsgType |  | 否 | 消息的类型，此时固定为event |
| Event |  | 否 | 事件的类型，此时固定为change_contact |
| ChangeType |  | 否 | 固定为update_tag |
| TagId |  | 否 | 标签Id |
| AddUserItems |  | 否 | 标签中新增的成员userid列表，用逗号分隔 |
| DelUserItems |  | 否 | 标签中删除的成员userid列表，用逗号分隔 |
| AddPartyItems |  | 否 | 标签中新增的部门id列表，用逗号分隔 |
| DelPartyItems |  | 否 | 标签中删除的部门id列表，用逗号分隔 |

## 其他说明

### 标签成员变更事件

该事件会回调给通讯录同步助手以及代开发自建应用
xml请求示例：
<xml><ToUserName><![CDATA[toUser]]></ToUserName><FromUserName><![CDATA[sys]]></FromUserName><CreateTime>1403610513</CreateTime><MsgType><![CDATA[event]]></MsgType><Event><![CDATA[change_contact]]></Event><ChangeType><![CDATA[update_tag]]></ChangeType><TagId>1</TagId><AddUserItems><![CDATA[zhangsan,lisi]]></AddUserItems><DelUserItems><![CDATA[zhangsan1,lisi1]]></DelUserItems><AddPartyItems><![CDATA[1,2]]></AddPartyItems><DelPartyItems><![CDATA[3,4]]></DelPartyItems></xml>
参数说明：
参数说明ToUserName企业微信CorpIDFromUserName此事件该值固定为sys，表示该消息由系统生成CreateTime消息创建时间 （整型）MsgType消息的类型，此时固定为eventEvent事件的类型，此时固定为change_contactChangeType固定为update_tagTagId标签IdAddUserItems标签中新增的成员userid列表，用逗号分隔DelUserItems标签中删除的成员userid列表，用逗号分隔AddPartyItems标签中新增的部门id列表，用逗号分隔DelPartyItems标签中删除的部门id列表，用逗号分隔
说明：由通讯录同步助手通过api发起的标签变更触发的事件不回调给通讯录同步助手应用。备注：标签成员变更事件与成员变更事件的时序不保证先后，如果需要严格依赖标签成员变更事件维护标签成员，可在收到标签成员变更事件时调用获取标签成员接口，以此获取标签成员。
