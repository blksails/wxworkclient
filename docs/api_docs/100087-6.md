## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 自建应用 | 自建应用 | 否 | 需具备「数据与智能专区权限」 |
| 代开发应用 | 代开发应用 | 否 | 需具备「数据与智能专区权限」 |
| 第三方应用 | 第三方应用 | 否 | 需具备「数据与智能专区权限」 |
| program_id |  | 是 | 应用关联的程序id |
| debug_token |  | 是 | 程序的调试凭证 |

#### 示例 1: 请求包体：


### 响应参数

| 参数名 | 类型 | 说明 |

| errmsg |  | 错误码说明 |

## 其他说明

### 应用开启调试模式

应用通过接口将指定的程序开启调试模式
权限说明
应用类型权限要求自建应用需具备「数据与智能专区权限」代开发应用需具备「数据与智能专区权限」第三方应用需具备「数据与智能专区权限」
对于第三方应用，仅通过测试安装的企业可开启调试模式对于代开发应用，企业必须为服务商的测试企业
请求方式：POST（HTTPS）请求地址：https://qyapi.weixin.qq.com/cgi-bin/chatdata/open_debug_mode?access_token=ACCESS_TOKEN
请求包体：
{"program_id":"xxx","debug_token":"xxx"}
参数说明：
参数是否必须说明program_id是应用关联的程序iddebug_token是程序的调试凭证
返回结果：
{"errcode":0,"errmsg":"ok"}
参数说明：
参数说明errcode错误码errmsg错误码说明
