# 创建预约会议

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93706](https://developer.work.weixin.qq.com/document/path/93706)
- **文档 ID**: `93706`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/create?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于创建一个预约会议。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| admin_userid | string | 是 | 会议管理员userid |
| title | string | 是 | 会议的标题，最多支持40个字节或者20个utf8字符 |
| meeting_start | uint32 | 是 | 会议开始时间的unix时间戳。需大于当前时间 |
| meeting_duration | uint32 | 是 | 会议持续时间（单位秒），最小300秒，最大86399秒 |
| description | string | 否 | 会议的描述，最多支持500个字节或者utf8字符 |
| location | string | 否 | 会议地点,最多128个字符 |
| agentid | uint32 | 否 | 授权方安装的应用agentid。仅旧的第三方多应用套件需要填此参数 |
| invitees.userid | string | 否 | 参与会议的企业成员userid |
| cal_id | string | 否 | 会议所属日历ID。该日历必须是access_token所对应应用所创建的日历。注意，若参与人在日历分享范围内，则插入到该日历（同时会插入会议参与人的默认日历），若不在分享范围内，否则仅插入到参与者默认日历； 如果不填，那么插入到参与者的默认日历上。 第三方应用必须指定cal_id 不多于64字节 |
| settings.password | string | 否 | 入会密码，仅可输入4-6位纯数字 |
| settings.enable_waiting_room | bool | 否 | 是否开启等候室。true:开启等候室；false:不开启等候室；默认不开 |
| settings.allow_enter_before_host | bool | 否 | 是否允许成员在主持人进会前加入。true:允许；false:不允许。默认允许 |
| settings.remind_scope | uint32 | 否 | 会议开始时来电提醒方式，1.不提醒 2.仅提醒主持人 3.提醒所有成员入 4.指定部分人响铃。默认仅提醒主持人 |
| settings.enable_enter_mute | uint32 | 否 | 成员入会时静音；1:开启；0:关闭；2:超过6人后自动开启静音。默认超过6人自动开启静音 |
| settings.enable_screen_watermark | bool | 否 | 是否开启屏幕水印，true:开启；false:不开启。默认不开启 |
| settings.hosts.userid | string | 否 | 企业成员userid |
| settings.ring_users.userid | string | 否 | 指定响铃的成员userid |
| reminders.is_repeat | uint32 | 否 | 是否是周期性会议，1：周期性会议 0：非周期性会议。默认为0 |
| reminders.repeat_type | uint32 | 否 | 周期性会议重复类型，0.每天；1.每周；2.每月；7.每个工作日。默认为0。周期性会议该字段才生效 |
| reminders.repeat_until | uint32 | 否 | 重复结束时刻。周期性会议该字段才生效。若会议结束时间超出最大结束时间或者未设置，则默认设置为最大结束时间。每天\每个工作日\每周 最多重复200次会议；每两周\每月最多重复50次会议 |
| reminders.repeat_interval | uint32 | 否 | 重复间隔。目前仅当repeat_type为1时，即周期为周时，支持设置该字段，且值不能大于2。 |
| reminders.remind_before | array | 否 | 指定会议开始前多久提醒成员，相对于meeting_start前的秒数，默认不提醒。目前仅支持 0：会议开始时提醒；300:5分钟前提醒；900:15分钟前提醒；3600:一小时前提醒；86400一天前提醒。若指定了非支持的值，则表现为会议开始时提醒 |

### 请求示例

```json
{
	"admin_userid": "zhangsan",
	"title": "新建会议",
	"meeting_start": 1600000000,
	"meeting_duration": 3600,
	"description": "新建会议描述",
	"location":"广州媒体港",
	"agentid": 1000014,
	"invitees": {
		"userid": ["lisi", "wangwu"]
	},
	"settings": {
		"remind_scope": 1,
		"password": "1234",
		"enable_waiting_room": false,
		"allow_enter_before_host": true,
		"enable_enter_mute": 1,
		"enable_screen_watermark": false,
		"hosts": {
			"userid": ["lisi", "wangwu"]
		},
		"ring_users": {
			"userid": ["zhangsan", "lis"]
		}
	},
	"cal_id": "wcjgewCwAAqeJcPI1d8Pwbjt7nttzAAA",
	"reminders": {
		"is_repeat": 1,
		"repeat_type": 0,
		"repeat_until": 1606976813,
		"repeat_interval": 1,
		"remind_before": [0, 900]
	}
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| meetingid | string | 会议id，通过此id可调用“进入会议”接口(包括小程序接口和JS-SDK接口)，以向参会人提供便捷入会入口 |
| excess_users | array | 参会人中包含无效会议账号的userid，仅在购买会议专业版企业由于部分参会人无有效会议账号时返回 |

### 响应示例

```json
{
   "errcode": 0,
   "errmsg": "ok",
   "meetingid": "XXXXXXXXX",
   "excess_users":["zhangsan","lisi"]
}
```
