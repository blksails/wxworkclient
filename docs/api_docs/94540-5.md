# 获取事件类别列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94540](https://developer.work.weixin.qq.com/document/path/94540)
- **文档 ID**: `94540`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/report/grid/list_cata?access_token=ACCESS_TOKEN`

## 接口描述

可以获取已配置的一级或二级事件类别，网格员在提交「巡查上报」或「居民上报」时将填入配置的事件类别

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证。“获取access_token” |
| 自建应用 | 自建应用 | 否 | 配置到「居民联系 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 支持，需要勾选「居民联系权限」 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| cata_list |  | 分类列表 |
| cata_list.category_id |  | 分类id |
| cata_list.category_name |  | 分类名称 |
| cata_list.level |  | 分类层级 |
| cata_list.parent_category_id |  | 分类层级为1时，该字段为空 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","category_list":[{"category_id":"category_id","category_name":"2222","level":1},{"category_id":"category_id","category_name":"2222","level":2,"parent_category_id":"parent_category_id"}]}
```
