# 获取命中关键词规则的会话记录

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/100029](https://developer.work.weixin.qq.com/document/path/100029)
- **文档 ID**: `100029`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/chatdata/get_hit_msg_list`

## 接口描述

通过该接口，可以获取一段时间内命中关键词规则或关键行为的会话记录。

## 请求信息

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| cursor | string | 否 | 上一次调用时返回的next_cursor，第一次拉取可以不填。若不填，从5天内最早的消息开始返回。不多于64字节 |
| token | string | 否 | 回调事件返回的 `token` 字段，10分钟内有效；建议都从回调事件中取出token填上，否则接口会有严格的频率限制。不多于128字节 |
| need_detail | bool | 否 | 是否需要消息详情，默认为否 |
| limit | int32 | 否 | 每次调用返回的记录数，默认为200，最大不超过1000 |
| begin_time | string | 否 | 仅用于需要重新拉取数据而不清楚具体cursor的场景，正常情况不应该使用该字段。会从begin_time附近的数据开始返回。 |

### 请求示例

```json
{
	"cursor":"RMTJID",
	"token":"NGEJKGOEGJKOEGNEOAGOEGOKEGHOEEEEE",
	"need_detail":1,
	"limit":200
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| has_more | bool | 是否还有更多数据。0-否；1-是。 |
| next_cursor | string | 下次调用带上该值，则从当前的位置继续往后拉，以实现增量拉取。强烈建议对该字段入库保存，每次请求读取带上，请求结束后更新。避免因意外丢，导致必须从头开始拉取，引起消息延迟。 |
| msg_list | object[] | 客户会话列表 |
| msg_list[].msgid | string | 每条消息对应的msgid |
| msg_list[].msg_detail | object | 消息详情，详见 MsgDetail。`仅need_detail=1时返回` |
| msg_list[].hit_rule_list | object[] | 命中的关键词规则列表 |
| msg_list[].hit_rule_list[].rule_id | string | 命中的规则id |
| msg_list[].hit_rule_list[].has_hit_keyword | bool | 是否命中该规则的关键词列表 |
| msg_list[].hit_rule_list[].semantics_list | int32[] | 该规则中命中的关键行为列表，1：红包；2：手机号码；3：邮箱地址；4：微信名片；5：带二维码图片；6：撤回消息；7：银行卡号；8：身份证号；9-发送文件（不包括微盘文件）；10-发送链接（发送链接消息或者发送的文本消息中包含链接）；11-发送小程序；12-发送客户欢迎语 |
