# 批量获取申请单ID - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99883](https://developer.work.weixin.qq.com/document/path/99883)
- **文档 ID**: `99883`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/advanced_feature/get_apply_id_list?access_token=ACCESS_TOKEN`

## 接口描述

自建应用可使用此接口，批量获取企业成员高级功能申请单的id信息。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| business_type | uint32 | 是 | 申请的高级账号类型1-邮件 2-文档 3-微盘 4-会议 |
| userid | string | 是 | 申请的userid |
| limit | uint32 | 否 | 分页查询的数据上限。默认100，最大200注意：不保证每次返回的数据刚好为指定limit，须用返回的has_more判断是否继续请求 |
| cursor | string | 否 | 用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |
| req_type | uint32 | 否 | 0-所有 1-仅api单 2-非api 申请单， 默认为0 |
| 自建应用 | 自建应用 | 否 | 配置到「高级功能-成员申请管理-通过api设置申请流程」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"business_type":1,"limit":10,"cursor":"CURSOR","req_type":1}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 错误码 |
| errmsg |  | 错误码说明 |
| next_cursor |  | 分页请求的下一页游标 |
| apply_id_list |  | 申请id列表 |
| has_more |  | 是否还有下一页数据 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","next_cursor":"CURSOR","apply_id_list":["APPLY_ID1","APPLY_ID2"],"has_more":true}
```
