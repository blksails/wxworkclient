# 观看直播回放 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93826](https://developer.work.weixin.qq.com/document/path/93826)
- **文档 ID**: `93826`

## 接口描述

此接口将调起直播间回放页面

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| livingId | string | 是 | 直播ID |
| success | Function | 否 | 接口调用成功的回调函数 |
| fail | Function | 否 | 接口调用失败的回调函数 |
| complete | Function | 否 | 接口调用结束的回调函数（调用成功、失败都会执行） |

### 请求示例

```json
wx.qy.replayLiving({"livingId":"LIVINGID",fail:function(res){console.log(res)},});
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| res.errMsg | string | 错误信息，如果调用成功返回 ${apiName}:ok |
| qy.replayLiving:ok |  | 执行成功 |
| qy.replayLiving:fail no session or expired |  | 没有调用 wx.qy.login，或者 session_key 已过期 |
| qy.replayLiving:fail no permission |  | 应用需具有直播权限 |
| qy.replayLiving:fail user not in allow list |  | 当前成员不在应用可见范围 |
| qy.replayLiving:fail invalid living id |  | 不合法的直播ID |
| qy.replayLiving:fail not allow to cross corp |  | 不可跨企业使用直播ID |
| qy.replayLiving:fail not allow to cross app |  | 不可跨应用使用直播ID |
| qy.replayLiving:fail living has no replay |  | 不存在直播回放 |
| qy.replayLiving:fail replay is beging creating |  | 正在直播中，或回放正在生成中，稍后观看回放 |
| qy.replayLiving:fail create replay failed |  | 回放创建失败 |
| qy.replayLiving:fail invalid parameter |  | 参数不合法 |

## 其他说明

### 权限说明

此接口仅在企业微信3.1.0及以后版本支持必须先调用过wx.qy.login，且session_key未过期，开发者可调用checkSession 检查当前登录态当前成员必须在应用的可见范围应用需具有直播使用权限，见配置可使用直播的应用

### 示例代码

wx.qy.replayLiving({"livingId":"LIVINGID",fail:function(res){console.log(res)},});
