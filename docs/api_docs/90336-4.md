# 获取部门成员

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/90336](https://developer.work.weixin.qq.com/document/path/90336)
- **文档 ID**: `90336`
- **请求方法**: `GET`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/user/simplelist`

## 接口描述

企业通讯录安全特别重要，企业微信将持续升级加固通讯录接口的安全机制。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |
| department_id | string | 是 | 获取的部门id |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| userlist | object[] | 成员列表 |
| userlist[].userid | string | 成员UserID。对应管理端的账号 |
| userlist[].name | string | 成员名称，代开发自建应用需要管理员授权才返回；此字段从2019年12月30日起，对新创建第三方应用不再返回真实name，使用userid代替name，2020年6月30日起，对所有历史第三方应用不再返回真实name，使用userid代替name，后续第三方仅通讯录应用可获取，未返回名称的情况需要通过通讯录展示组件来展示名字 |
| userlist[].department | int32[] | 成员所属部门列表。列表项为部门ID，32位整型 |
| userlist[].open_userid | string | 全局唯一。对于同一个服务商，不同应用获取到企业内同一个成员的open_userid是相同的，最多64个字节。仅第三方应用可获取 |

## 其他说明

### 权限说明

应用须拥有指定部门的查看权限。
