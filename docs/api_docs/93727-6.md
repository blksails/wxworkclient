# 获取对外收款记录

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93727](https://developer.work.weixin.qq.com/document/path/93727)
- **文档 ID**: `93727`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/externalpay/get_bill_list`

## 接口描述

企业和服务商可通过此接口获取企业的对外收款记录。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 访问令牌 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| begin_time | int32 | 是 | 收款记录开始时间戳，单位为秒 |
| end_time | int32 | 是 | 收款记录结束时间戳，单位为秒 |
| payee_userid | string | 否 | 企业收款成员userid，不填则为全部成员 |
| cursor | string | 否 | 用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |
| limit | int32 | 否 | 返回的最大记录数，整型，最大值1000 |

### 请求示例

```json
{
   "begin_time":1605171726,
   "end_time":1605172726,
   "payee_userid":"zhangshan",
   "cursor":"CURSOR",
   "limit":10
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 分页游标，在下次请求时填写以获取之后分页的记录，如果已经没有更多的数据则不返回该字段 |
| bill_list | object[] | 交易单详情列表 |
| bill_list[].transaction_id | string | 交易单号 |
| bill_list[].bill_type | int32 | 交易类型： 0：收款记录 1：退款记录 |
| bill_list[].trade_state | int32 | 交易状态（退款记录不返回该字段）： 1：已完成  3：已完成有退款 |
| bill_list[].pay_time | int32 | 交易时间 |
| bill_list[].out_trade_no | string | 商户单号。如果是退款记录，返回对应收款记录的商户单号 |
| bill_list[].out_refund_no | string | 退款单号，退款记录返回该字段 |
| bill_list[].external_userid | string | 付款人的userid |
| bill_list[].total_fee | int32 | 收款总金额，单位为分 |
| bill_list[].payee_userid | string | 企业内账号userid。退款记录的该字段为退款成员userid，收款记录的该字段为收款成员userid |
| bill_list[].payment_type | int32 | 收款方式： 0：在聊天中收款 1：收款码收款  2：在直播间收款 3：用产品图册收款 14：转账 15：小程序（仅部分灰度企业可在小程序中接入对外收款账户进行收款） |
| bill_list[].mch_id | string | 收款商户号id |
| bill_list[].remark | string | 备注。退款记录的该字段为退款备注，收款记录的该字段为收款备注 |
| bill_list[].commodity_list | object[] | 商品信息详情列表 |
| bill_list[].total_refund_fee | int32 | 退款总金额 |
| bill_list[].refund_list | object[] | 退款单据详情列表 |
| bill_list[].contact_info | object | 联系人信息 |
| bill_list[].miniprogram_info | object | 小程序信息 |

## 其他说明

### 补充说明

收款记录的起止时间间隔不能超过1个月。会过滤收款人不在可见范围中的收款记录，因此返回的记录数可能会小于limit设置的最大记录数。如果没有获取到next_cursor，说明已经拉取完所有的数据。

### 权限说明

只会返回当前在应用可见范围内用户的收款记录。4.1.0及以上版本中新增的部分商户号收款记录，不会返回给第三方应用和代开发应用。调用的应用需要满足如下的权限：
- 自建应用：配置到「对外收款- 可调用接口的应用」中
- 代开发应用：具有「对外收款」权限
- 第三方应用：具有「对外收款」权限
从2023年12月1日0点起，不再支持通过系统应用secret调用接口，存量企业暂不受影响 查看详情
