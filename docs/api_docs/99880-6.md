# 设置审批单审批信息 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99880](https://developer.work.weixin.qq.com/document/path/99880)
- **文档 ID**: `99880`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/advanced_feature/set_approval_detail?access_token=ACCESS_TOKEN`

## 接口描述

自建应用可使用此接口，设置企业成员高级功能申请单的审批信息。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| apply_id | string | 是 | 申请id |
| approval_id | string | 是 | 审批id，注意：应用生成审批id后，审批id和申请id是一一对应的，不可改变 |
| approval_status | uint32 | 是 | 审批单状态：1-审批中; 2-已驳回; 3-已同意; 101-已撤销 |
| approval_url | string | 是 | 审批单跳转链接，须已"http://"或"https://"开头 |
| 自建应用 | 自建应用 | 否 | 配置到「高级功能-成员申请管理-通过api设置申请流程」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"approval_id":"APPROVALID","approval_status":1,"apply_id":"ApplyId","approval_url":"www.qq.com","process_list":{"node_list":[{"current_approvers":["LEADER1","LEADER2"],"completed_approvers":["LEADER0"],"node_apv_status":1,"node_apv_rel":1,"apv_update_time":1715741759},{"current_approvers":["LEADER3","LEADER4"],"completed_approvers":["LEADER5"],"node_status":1,"node_apv_rel":1,"apv_update_time":1715741759}]}}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| process_list.node_list | array(obj) | 审批单审批节点，注意：如果需要变更审批节点信息，需要全量节点都传入 |
| process_list.node_list.node_apv_status | uint32 | 审批节点状态：1-审批中; 2-已驳回; 3-已同意; 101-已撤销; 102-未到流程 |
| process_list.node_list.node_apv_rel | uint32 | 审批节点多人审批方式：1-会签；2-或签；3-依次审批 |
| process_list.node_list.current_approvers | array(string ) | 当前审批节点待处理人列表，最多100个，待处理人列表和已处理人列表不可同时为空 |
| process_list.node_list.completed_approvers | array(string) | 当前审批节点已处理人列表，做多100个，待处理人列表和已处理人列表不可同时为空 |
| process_list.node_list.apv_update_time | uint32 | 审批节点更新时间 |
| errcode |  | 错误码 |
| errmsg |  | 错误码说明 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
