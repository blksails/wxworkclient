# 修改会议高级布局 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98868](https://developer.work.weixin.qq.com/document/path/98868)
- **文档 ID**: `98868`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/advanced_layout/update?access_token=ACCESS_TOKEN`

## 接口描述

对会议中的高级布局进行修改，注意修改的是布局定义。若修改的会议布局正被会议使用，新布局会自动应用到会议。若修改的会议布局正在被用户使用，新布局不会自动应用到用户。接口仅支持全量更新，不支持部分字段单独更新。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 是 | 会议 ID |
| layout_id | string | 是 | 布局 ID |
| layout_name | string | 否 | 布局名称 |
| page_list | object[] | 是 | 布局单页对象列表，详见LayoutPage |
| layout_template_id | string | 是 | 布局模板 ID |
| enable_polling | bool | 是 | 开启或关闭轮询，默认关闭 |
| polling_setting | object | 否 | 轮询参数设置对象，详见PollingSetting |
| user_seat_list | object[] | 否 | 用户座次对象列表，详见UserSeat |
| polling_interval_unit | uint32 | 是 | 轮询开启后设置参数。轮询间隔时间类型：1：秒2：分钟 |
| polling_interval | uint32 | 是 | 轮询开启后设置参数 轮询间隔时长，允许取值范围1～999999 |
| ignore_user_novideo | bool | 是 | 轮询开启后设置参数，设置是否忽略没开启视频成员 |
| ignore_user_absence | bool | 是 | 轮询开启后设置参数，设置是否忽略未入会成员 |
| grid_id | string | 是 | 宫格 ID |
| grid_type | uint32 | 是 | 宫格类型：1. 视频画面2. 共享画面 |
| video_type | uint32 | 否 | 视频画面来源：1：演讲者2：自动填充3：指定人员。根据 user_list 的定义显示视频内容（此类型需传递 tmp_openid）。 |
| user_list | object[] | 否 | 宫格中的用户列表：轮询关闭，只有一个用户；轮询开启后，可以包含多个用户。详见GridUser |
| userid | string | 否 | 本场会议企业成员的userid。userid与tmp_openid必填其一 |
| tmp_openid | string | 否 | 用户当前会议临时身份 ID，单场会议唯一 |

### 请求示例

```json
{"meetingid":"MEETINGID","layout_id":"LAYOUTID1","layout_name":"testmictest","page_list":[{"layout_template_id":"3","enable_polling":true,"polling_setting":{"polling_interval_unit":1,"polling_interval":10,"ignore_user_novideo":false,"ignore_user_absence":false},"user_seat_list":[{"grid_id":"1","grid_type":1,"video_type":3,"user_list":[{"userid":"USERID","tmp_openid":"TMPOPENID1"}]}]}]}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
