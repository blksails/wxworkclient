# 获取应用二维码

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95430](https://developer.work.weixin.qq.com/document/path/95430)
- **文档 ID**: `95430`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/service/get_app_qrcode?suite_access_token=SUITE_ACCESS_TOKEN`

## 接口描述

用于获取第三方应用二维码。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| suite_access_token | string | 是 | 第三方应用access_token，获取方法参见 获取第三方应用凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| suite_id | string | 是 | 第三方应用id（即ww或wx开头的suiteid） |
| appid | int32 | 否 | 第三方应用id，单应用不需要该参数，多应用旧套件才需要传该参数。若不传默认为1 |
| state | string | 否 | state值，用于区分不同的安装渠道，可以填写a-zA-Z0-9，长度不可超过32个字节，默认为空。扫应用带参二维码授权安装后，获取企业永久授权码接口会返回该state值 |
| style | int32 | 否 | 二维码样式选项，默认为不带说明外框小尺寸。0：带说明外框的二维码，适合于实体物料，1：带说明外框的二维码，适合于屏幕类，2：不带说明外框（小尺寸），3：不带说明外框（中尺寸），4：不带说明外框（大尺寸）。具体样式与服务商管理端获取到的应用二维码样式一一对应，参见下文二维码样式说明 |
| result_type | int32 | 否 | 结果返回方式，默认为返回二维码图片buffer。1：二维码图片buffer，2：二维码图片url |

### 请求示例

```json
{
	"suite_id":"xxx",
	"appid":1,
	"state":"test",
	"style":0,
	"result_type":1
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误描述 |
| qrcode | string | 二维码url地址 |

## 其他说明

### 权限说明

要求第三方应用是已上线的第三方通用应用

### 二维码样式说明

适合于实体物料的带说明外框的二维码
适合于屏幕类带说明外框的二维码
不带说明的二维码，有三种不同尺寸
小尺寸：312×384，最佳扫描距离：0.5m，适用场景：屏幕类、宣传册等
中尺寸：624×768，最佳扫描距离：1m，适用场景：海报、展架等
大尺寸：1996×2456，最佳扫描距离：2.5m，适用场景：幕布、大型广告等
