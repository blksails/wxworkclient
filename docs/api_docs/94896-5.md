# 唤起选择企业通讯录成员

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94896](https://developer.work.weixin.qq.com/document/path/94896)
- **文档 ID**: `94896`
- **请求方法**: `wx.qy.selectPrivilegedContact`
- **接口地址**: `wx.qy.selectPrivilegedContact`

## 接口描述

用于第三方应用唤起选择企业通讯录成员，用户选择的范围区分成两部分回传给第三方应用。

## 请求信息

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| fromDepartmentId | Int32 | 是 | 表示打开的通讯录从指定的部门开始展示，-1表示自己所在部门开始, 0表示从最上层开始 |
| mode | String | 是 | 选择模式，single表示单选，multi表示多选 |
| selectedContextContact | Int32 | 否 | 是否勾选当前环境的参与者。1表示是，0表示否。目前仅支持“群加号”的场景。开发者可根据getContext接口判断所在环境，目前仅支持chat_attachment场景值 |
| selectedOpenUserIds | 列表 | 否 | 已选用户openUserId列表。仅支持multi模式，若传了single，忽略该参数。若传入的openuserid非法，忽略。列表不超过2000 |
| selectedTickets | 列表 | 否 | 已选ticket列表。仅支持multi模式，若传了single，忽略该参数。最多传入10个ticket |

### 请求示例

```text
wx.qy.selectPrivilegedContact( {
    "fromDepartmentId": -1,
    "mode": "multi",
    "selectedContextContact": 1,
    "selectedOpenUserIds": ["xxx","yyy"],
    "selectedTickets": ["ticket1","ticket2"],
    success: function(res){
        var selectedUserList = res.result.userList;
        for (var i = 0; i < selectedUserList.length; i++)
        {
            var openUserId = selectedUserList[i].openUserId;
        }
        var selectedTicket = res.result.selectedTicket;
        var expiresIn = res.result.expiresIn;
        var selectedUserCount = res.result.selectedUserCount;
    }
});
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| err_msg | String | 调用成功时返回selectPrivilegedContact:ok 若当前环境不支持选择，且传值了selectedContextContact=1，返回selectPrivilegedContact:fail_unsupported_context 若传了selectedContextContact=1，且mode=single时，返回 selectPrivilegedContact:fail_context_require_multi_mode |
| result | SelectResult | 当调用成功时返回，对应用户选择的内容 |
