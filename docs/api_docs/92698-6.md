# 创建企业群发

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/92698](https://developer.work.weixin.qq.com/document/path/92698)
- **文档 ID**: `92698`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/externalcontact/add_msg_template`

## 接口描述

企业跟第三方应用可通过此接口添加企业群发消息的任务并通知成员发送给相关客户或客户群。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chat_type | string | 否 | 群发任务的类型，默认为single，表示发送给客户，group表示发送给客户群 |
| external_userid | string[] | 否 | 客户的 externaluserid 列表，仅在 chat_type 为 single 时有效，最多可一次指定 1 万个客户 |
| chat_id_list | string[] | 否 | 客户群 id 列表，仅在 chat_type 为 group 时有效，最多可一次指定 2000 个客户群。指定群 id 之后，收到任务的群主无须再选择客户群，仅对 4.1.10 及以上版本的企业微信终端生效 |
| tag_filter.group_list.tag_list | string[] | 否 | 要进行群发的客户标签列表，同组标签之间按或关系进行筛选，不同组标签按且关系筛选，每组最多指定 100 个标签，支持规则组标签 |
| sender | string | 否 | 发送企业群发消息的成员 userid，当类型为发送给客户群时必填 |
| allow_select | bool | 否 | 是否允许成员在待发送客户列表中重新进行选择，默认为 false，仅支持客户群发场景 |
| text.content | string | 否 | 消息文本内容，最多 4000 个字节 |
| attachments | object[] | 否 | 附件，最多支持添加 9 个附件 |
| attachments.msgtype | string | 是 | 附件类型，可选 image、link、miniprogram、video 或者 file |
| image.media_id | string | 否 | 图片的 media_id，可以通过素材管理接口获得 |
| image.pic_url | string | 否 | 图片的链接，仅可使用上传图片接口得到的链接 |
| link.title | string | 是 | 图文消息标题，最长 128 个字节 |
| link.picurl | string | 否 | 图文消息封面的 url，最长 2048 个字节 |
| link.desc | string | 否 | 图文消息的描述，最多 512 个字节 |
| link.url | string | 是 | 图文消息的链接，最长 2048 个字节 |
| miniprogram.title | string | 是 | 小程序消息标题，最多 64 个字节 |
| miniprogram.pic_media_id | string | 是 | 小程序消息封面的 mediaid，封面图建议尺寸为 520*416 |
| miniprogram.appid | string | 是 | 小程序 appid（可以在微信公众平台上查询），必须是关联到企业的小程序应用 |
| miniprogram.page | string | 是 | 小程序 page 路径 |
| video.media_id | string | 是 | 视频的 media_id，可以通过素材管理接口获得 |
| file.media_id | string | 是 | 文件的 media_id，可以通过素材管理接口获得 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| fail_list | string[] | 无效或无法发送的 external_userid 或 chatid 列表 |
| msgid | string | 企业群发消息的 id，可用于获取群发消息发送结果 |

### 响应示例

```json
{
    "errcode": 0,
    "errmsg": "ok",
	"fail_list":["wmqfasd1e1927831123109rBAAAA"],
	"msgid":"msgGCAAAXtWyujaWJHDDGi0mAAAA"
}
```
