# 自动激活回调通知 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95994](https://developer.work.weixin.qq.com/document/path/95994)
- **文档 ID**: `95994`

## 接口描述

企业成员满足自动激活条件并触发自动激活后，企业微信后台会推送“自动激活通知”到服务商的系统事件接收URL（应用管理 -通用开发参数-系统事件接收URL）。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ServiceCorpId |  | 否 | 服务商CorpID |
| InfoType |  | 否 | 事件类型，此时固定为auto_activate |
| AuthCorpId |  | 否 | 客户企业CorpID |
| Scene |  | 否 | 许可自动激活的时机，1:企业成员主动访问应用，2:服务商调用消息推送接口，3:服务商调用互通接口 |
| TimeStamp |  | 否 | 时间戳 |
| AccountList |  | 否 | 激活的许可账号列表 |
| ActiveCode |  | 否 | 自动激活的许可账号激活码 |
| Type |  | 否 | 自动激活的许可的类型，1:基础许可，2:互通许可 |
| ExpireTime |  | 否 | 自动激活后，该许可的到期时间 |
| UserId |  | 否 | 许可自动激活的成员的UserID |
| PreviousStatus |  | 否 | 激活成员自动激活前的许可状态，1:未激活许可，2:已激活许可且许可未过期（即许可的剩余时长小于等于7天），3:已激活许可且许可已过期 |

### 请求示例

```xml
<xml><ServiceCorpId><![CDATA[aaaa]]></ServiceCorpId><InfoType><![CDATA[auto_activate]]></InfoType><AuthCorpId><![CDATA[bbbb]]></AuthCorpId><Scene>1</Scene><TimeStamp>1403610513</TimeStamp><AccountList><ActiveCode><![CDATA[XXXX]]></ActiveCode><Type>1</Type><ExpireTime>1700000000</ExpireTime><UserId><![CDATA[USERID]]></UserId><PreviousStatus>1</PreviousStatus><PreviousActiveCode><![CDATA[YYYY]]></PreviousActiveCode></AccountList></xml>
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| PreviousActiveCode |  | 仅针对已激活的成员进行自动激活时返回，返回该成员之前激活的旧的激活码 |
