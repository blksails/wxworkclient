# 增量更新成员

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/91130](https://developer.work.weixin.qq.com/document/path/91130)
- **文档 ID**: `91130`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/batch/syncuser`

## 接口描述

本接口以userid（账号）为主键，增量更新企业微信通讯录成员。请先下载CSV模板，根据需求填写文件内容。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 访问令牌 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| media_id | string | 是 | 上传的csv文件的media_id |
| to_invite | bool | 否 | 是否邀请新建的成员使用企业微信，默认值为true |
| callback.url | string | 否 | 企业应用接收企业微信推送请求的访问协议和地址，支持http或https协议 |
| callback.token | string | 否 | 用于生成签名 |
| callback.encodingaeskey | string | 否 | 用于消息体的加密，是AES密钥的Base64编码 |

### 请求示例

```json
{
	"media_id":"xxxxxx",
	"to_invite": true,
	"callback":
	{
	 	"url": "xxx",
	 	"token": "xxx",
	 	"encodingaeskey": "xxx"
	}
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| jobid | string | 异步任务id，最大长度为64字节 |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"jobid": "xxxxx"
}
```

## 其他说明

### 注意事项

* 模板中的部门需填写部门ID，多个部门用分号分隔，部门ID必须为数字，根部门的部门id默认为1。当填写的部门ID不存在时，会在其他（待设置部门下）新建此部门。
* 文件中存在、通讯录中也存在的成员，更新成员在文件中指定的字段值
* 文件中存在、通讯录中不存在的成员，执行添加操作
* 通讯录中存在、文件中不存在的成员，保持不变
* 成员字段更新规则：可自行添加扩展字段。文件中有指定的字段，以指定的字段值为准；文件中没指定的字段，不更新

### 权限说明

须拥有通讯录的写权限。
