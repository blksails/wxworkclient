# 为打卡人员排班 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/96501](https://developer.work.weixin.qq.com/document/path/96501)
- **文档 ID**: `96501`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/checkin/setcheckinschedulist?access_token=ACCESS_TOKEN`

## 接口描述

企业可通过具有调用权限的应用，为打卡规则为“按班次上下班”规则的指定员工排班。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证。获取方式参考：文档-获取access_token |
| items |  | 是 | 排班表信息 |
| groupid |  | 是 | 打卡规则的规则id，可通过“获取打卡规则”、“获取打卡数据”、“获取打卡人员排班信息”等相关接口获取 |
| userid |  | 是 | 打卡人员userid |
| day |  | 是 | 要设置的天日期，取值在1-31之间。联合yearmonth组成唯一日期 比如20201205 |
| schedule_id |  | 是 | 对应groupid规则下的班次id，通过预先拉取规则信息获取，0代表休息 |
| yearmonth |  | 是 | 排班表月份，格式为年月，如202011 |
| 自建应用 | 自建应用 | 否 | 配置到「打卡 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 具有「打卡」权限 |
| 第三方应用 | 第三方应用 | 否 | 具有「打卡」权限 |

### 请求示例

```json
{"groupid":226,"items":[{"userid":"james","day":5,"schedule_id":234}],"yearmonth":202012}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 错误码描述 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok"}
```
