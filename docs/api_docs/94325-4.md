## 接口描述

注册应用信息。

## 请求信息

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| options.corpId | string | 是 | 当前用户所属企业 ID |
| options.agentId | number | 否 | 当前应用 AgentID |
| options.suiteId | string | 否 | 应用套件 ID |
| options.jsApiList | string[] | 否 | 需要使用的JS接口列表 |
| options.openTagList | string[] | 否 | 需要使用的开放标签列表 |
| options.getConfigSignature | Function | 否 | config 签名生成函数 |
| options.onConfigSuccess | Function | 否 | config 成功回调 |
| options.onConfigFail | Function | 否 | config 失败回调 |
| options.onConfigComplete | Function | 否 | config 完成回调 |
| options.getAgentConfigSignature | Function | 否 | agentConfig 签名生成函数 |
| options.onAgentConfigSuccess | Function | 否 | agentConfig 成功回调 |
| options.onAgentConfigFail | Function | 否 | agentConfig 失败回调 |
| options.onAgentConfigComplete | Function | 否 | agentConfig 完成回调 |
| options.getSuiteConfigSignature | Function | 否 | suiteConfig 签名生成函数 |

### 请求示例

```text
ww.register({
  corpId: 'ww7ca4776b2a70000',
  jsApiList: ['getExternalContact'],
  getConfigSignature
})
```

## 其他说明

### 附加说明

参数中的回调函数（如 getConfigSignature、onConfigSuccess 等）调用时机由 JSSDK 自行控制，开发者无需关心具体的调用顺序。在非微信、企业微信内置浏览器的环境下，JSSDK 不会主动调用 getConfigSignature 回调。
