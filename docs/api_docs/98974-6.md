# 查询申请单状态 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98974](https://developer.work.weixin.qq.com/document/path/98974)
- **文档 ID**: `98974`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/miniapppay/get_applyment_status?access_token=ACCESS_TOKEN`

## 接口描述

请求方式：POST(HTTPS)请求地址：https://qyapi.weixin.qq.com/cgi-bin/miniapppay/get_applyment_status?access_token=ACCESS_TOKEN

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 自建应用 | 自建应用 | 否 | 配置到「对外收款- 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 暂不支持 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |
| access_token |  | 是 | 调用接口凭证 |
| out_request_no |  | 是 | 业务申请编号，长度限制为1~32个字符，在提交创建对外收款账户的申请单中提交 |
| applyment_state |  | 否 | 申请状态枚举值参考申请状态枚举值 |
| applyment_state_desc |  | 否 | 申请状态描述 |
| sign_state |  | 否 | 签约状态枚举值参考签约状态枚举值 |
| CHECKING |  | 否 | 待签约 |
| ACCOUNT_NEED_VERIFY |  | 否 | 完成 |
| AUDITING |  | 否 | 已冻结 |
| REJECTED |  | 否 | 已作废 |
| UNSIGNED |  | 否 | 未签约。该状态下，电商平台可查询获取签约链接，引导二级商户的超级管理员完成签约 |
| SIGNED |  | 否 | 已签约。指二级商户的超级管理员已完成签约。注意：若申请单被驳回，商户修改了商户主体名称、法人名称、超级管理员信息、主体类型等信息，则需重新签约 |
| NOT_SIGNABLE |  | 否 | 不可签约。该状态下，暂不支持超级管理员签约。一般为申请单处于已驳回、已冻结、机器校验中状态，无法签约 |
| account_name |  | 否 | 付款户名需商户使用该户名的账户进行汇款。 |
| remark |  | 否 | 备注信息商户汇款时，需要填写的备注信息。 |
| 0 |  | 否 | 绑定失败 |
| 1 |  | 否 | 未申请 |
| 2 |  | 否 | 待法人验证 |
| 3 |  | 否 |  |
| 0 |  | 否 | 不可签约 |
| 1 |  | 否 | 未签约 |
| 2 |  | 否 | 已签约 |

### 请求示例

```json
{"out_request_no":"xxxxx"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述 |
| status |  | 申请单的具体状态，参考申请单状态参数说明 |
| apply_state |  | 申请单当前所处阶段，对应关系参考申请阶段对应关系 |
| real_sign_state |  | 当前签约阶段，对应关系参考签约阶段对应关系 |
| reject_reason |  | 驳回理由 |
| sign_url |  | 签约链接1、当申请状态为NEED_SIGN 或 签约状态为UNSIGNED时返回，该链接为永久有效；2、申请单中的超级管理者，需用已实名认证的微信扫码打开，完成签约。 |
| sub_mchid |  | 电商平台二级商户号当申请状态为NEED_SIGN或FINISH时才返回。 |
| audit_detail |  | 驳回原因详情各项资料的审核情况。当申请状态为REJECTED或 FROZEN时才返回。 |
| audit_detail.param_name |  | 参数名称提交申请单的资料项名称。 |
| audit_detail.reject_reason |  | 驳回原因提交资料项被驳回原因。 |
| account_validation |  | 汇款账户验证信息当签约阶段字段（real_sign_state字段，注意不是status字段下的sign_state字段）的值为2（即已签约）时才有可能返回。当申请状态为ACCOUNT_NEED_VERIFY 时有返回，可根据指引汇款，完成账户验证。参考汇款账户验证信息参数说明 |
| legal_validation_url |  | 法人验证链接当签约阶段字段（real_sign_state字段，注意不是status字段下的sign_state字段）的值为2（即已签约）时才有可能返回。1、当申请状态为ACCOUNT_NEED_VERIFY，且通过系统校验的申请单，将返回链接。2、建议将链接转为二维码展示，让商户法人用微信扫码打开，完成账户验证。注：商户申请单进入审核状态后，微信侧会校验法人证件号码是否跟营业执照匹配，若匹配，返回该字段 ; 若不匹配，不支持法人扫码验证，不返回该字段。 |
| account_no |  | 付款卡号结算账户为对私时会返回，商户需使用该付款卡号进行汇款。 |
| pay_amount |  | 汇款金额需要汇款的金额(单位：分)。 |
| destination_account_number |  | 收款卡号收款账户的卡号 |
| destination_account_name |  | 收款户名destination_account_name |
| destination_account_bank |  | 开户银行收款账户的开户银行名称。 |
| city |  | 省市信息收款账户的省市。 |
| deadline |  | 汇款截止时间请在此时间前完成汇款。 |
