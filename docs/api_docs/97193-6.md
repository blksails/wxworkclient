# 分配激活码给下游/下级企业

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97193](https://developer.work.weixin.qq.com/document/path/97193)
- **文档 ID**: `97193`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/license/batch_share_active_code`

## 接口描述

服务商可调用该接口将为上游/上级企业购买的激活码分配给下游/下级企业使用。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| provider_access_token | string | 是 | 应用服务商的接口调用凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| from_corpid | string | 是 | 上游/上级企业corpid |
| to_corpid | string | 是 | 下游/下级企业corpid |
| share_list | object[] | 是 | 分配的接口许可列表。每次分配激活码不可超过1000个，且每次分配给下游/下级企业的激活码数不可超过上下游/企业互联通讯录中该下游企业人数的2倍 |
| share_list[].active_code | string | 是 | 分配的激活码 |
| corp_link_type | int32 | 否 | 分配的场景，不填默认为0 0：上下游 1：企业互联 |

### 请求示例

```json
{
	"from_corpid": "FROMCORPID",
	"to_corpid": "ToCORPID",
	"share_list":[
	{
		"active_code":"ACTIVE_CODE1"
	},
	{
	 	"active_code":"ACTIVE_CODE2"
	}],
	"corp_link_type": 0
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| share_result | object[] | 分配结果，仅分配失败的会返回 |
| share_result[].active_code | string | 分配的激活码 |
| share_result[].errcode | int32 | 错误码说明 |
| share_result[].errmsg | string | 错误描述 |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"share_result":[
	{
		"active_code": "ACTIVE_CODE",
		"errcode": 700100,
		"errmsg": "error"
	}]
}
```
