# 新增对接规则 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95634](https://developer.work.weixin.qq.com/document/path/95634)
- **文档 ID**: `95634`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/corpgroup/rule/add_rule?access_token=ACCESS_TOKEN`

## 接口描述

上下游系统应用可通过该接口新增一条对接规则。注意：新增和更新上下游对接规则的接口每天最多调用1000次

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| chain_id |  | 是 | 上下游id |
| rule_info |  | 是 | 上下游关系规则的详情 |
| 自建应用 | 自建应用 | 否 | 配置到「上下游- 可调用接口的应用」中 |

### 请求示例

```json
{"chain_id":"Chxxxxxx","rule_info":{"owner_corp_range":{"departmentids":["departmentid1","departmentid2"],"userids":["userid1","userid2"]},"member_corp_range":{"groupids":["groupid1","groupid2"],"corpids":["corpid1","corpid2"]}}}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| rule_info.owner_corp_range |  | 上游企业的对接人规则（下游企业可以看到并联系的成员或部门） |
| rule_info.owner_corp_range.departmentids |  | 部门id，部门id和用户id两个必选填一个 |
| rule_info.owner_corp_range.userids |  | 用户id，部门id和用户id两个必选填一个 |
| rule_info.member_corp_range |  | 下游企业规则范围 |
| rule_info.member_corp_range.groupids |  | 分组id，分组id和企业id两个必选填一个 |
| rule_info.member_corp_range.corpids |  | 企业id，分组id和企业id两个必选填一个 |
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| rule_id |  | 上下游规则id |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","rule_id":1}
```
