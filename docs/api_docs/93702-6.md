# 创建日历

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/93702](https://developer.work.weixin.qq.com/document/path/93702)
- **文档 ID**: `93702`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/oa/calendar/add`

## 接口描述

该接口用于通过应用在企业内创建一个日历。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 访问令牌 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| calendar.admins | string[] | 否 | 日历的管理员userid列表，管理员必须在通知范围成员的列表中。最多指定3人 |
| calendar.set_as_default | int32 | 否 | 是否将该日历设置为应用的默认日历。0-否；1-是。第三方应用不支持使用该参数 |
| calendar.summary | string | 是 | 日历标题。1 ~ 128 字符 |
| calendar.color | string | 是 | 日历在终端上显示的颜色，RGB颜色编码16进制表示，例如："#0000FF" 表示纯蓝色 |
| calendar.description | string | 否 | 日历描述。0 ~ 512 字符 |
| calendar.shares | object[] | 否 | 日历通知范围成员列表。最多2000人 |
| calendar.is_public | int32 | 否 | 是否公共日历。0-否；1-是。注意：每个人最多可创建或订阅100个公共日历。该属性不可更新 |
| calendar.public_range | object | 否 | 公开范围。仅当是公共日历时有效 |
| calendar.is_corp_calendar | int32 | 否 | 是否全员日历。0-否；1-是。注意：1. 每个企业最多可创建20个全员日历 2. 全员日历也是公共日历的一种，需要指定`public_range` 3. 全员日历不支持指定颜色、默认日历、只读权限 4. 该属性不可更新 |
| calendar.agentid | int32 | 否 | 授权方安装的应用agentid。仅旧的第三方多应用套件需要填此参数 |

### 请求示例

```json
{
	"calendar": {
		"admins":[
				"admin1",
				"admin2"
		],
		"set_as_default": 1,
		"summary": "test_summary",
		"color": "#FF3030",
		"description": "test_describe",
		"shares": [{
				"userid": "userid2",
				"permission": 1
			},
			{
				"userid": "userid3",
				"permission": 3
			}
		],
		"is_public": 1,
		"public_range": {
			"userids": ["abel", "jack"],
			"partyids": [1232, 34353]
		},
		"is_corp_calendar": 1
	},
	"agentid": 1000014
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| cal_id | string | 日历ID |
| fail_result | object | 无效的输入内容 |
| fail_result.shares | object[] | 无效的日历通知范围成员列表 |
| fail_result.shares.errcode | int32 | 错误码 |
| fail_result.shares.errmsg | string | 错误码说明 |
| fail_result.shares.userid | string | 日历通知范围成员的id |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg" : "ok",
	"cal_id":"wcjgewCwAAqeJcPI1d8Pwbjt7nttzAAA",
	"fail_result": {
		"shares":[{
			"errcode": 40001,
			"errmsg": "not found",
			"userid": "userid3"
		}]
	}
}
```
