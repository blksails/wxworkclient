# 获取老师直播ID列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97127](https://developer.work.weixin.qq.com/document/path/97127)
- **文档 ID**: `97127`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/living/get_user_all_livingid?access_token=ACCESS_TOKEN`

## 接口描述

通过此接口可以获取指定老师的所有直播ID

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| userid |  | 是 | 企业成员的userid |
| cursor |  | 否 | 上一次调用时返回的next_cursor，第一次拉取可以不填 |
| limit |  | 否 | 每次拉取的数据量，默认值和最大值都为100 |
| 自建应用 | 自建应用 | 否 | 配置到「上课直播/直播 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 具有「上课直播/直播」权限 |
| 第三方应用 | 第三方应用 | 否 | 具有「上课直播/直播」权限 |

### 请求示例

```json
{"userid":"USERID","cursor":"NEXT_KEY","limit":20}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| next_cursor |  | 当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取，返回空字符串代表已经是最后一页 |
| livingid_list |  | 直播ID列表 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","next_cursor":"next_cursor","livingid_list":["livingid1","livingid2"]}
```
