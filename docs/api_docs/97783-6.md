# 更新日历

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97783](https://developer.work.weixin.qq.com/document/path/97783)
- **文档 ID**: `97783`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/oa/calendar/update`

## 接口描述

该接口用于修改指定日历的信息。注意，更新操作是覆盖式，而不是增量式。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 访问令牌 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| skip_public_range | int32 | 否 | 是否不更新可订阅范围。0-否；1-是。默认会为0，会更新可订阅范围 |
| calendar.cal_id | string | 是 | 日历ID |
| calendar.admins | string[] | 否 | 日历的管理员userid列表。最多指定3人 |
| calendar.summary | string | 是 | 日历标题。1 ~ 128 字符 |
| calendar.color | string | 是 | 日历颜色，RGB颜色编码16进制表示，例如："#0000FF" 表示纯蓝色 |
| calendar.description | string | 否 | 日历描述。0 ~ 512 字符 |
| calendar.public_range.userids | string[] | 否 | 公开的成员列表范围 。最多指定1000个成员 |
| calendar.public_range.partyids | int32[] | 否 | 公开的部门列表范围 。最多指定100个部门 |
| calendar.shares | object[] | 否 | 日历通知范围成员列表。最多2000人 |
| calendar.shares.userid | string | 是 | 日历通知范围成员的id |
| calendar.shares.permission | int32 | 否 | 日历通知范围成员权限。1：可查看  3：仅查看闲忙状态 |

### 请求示例

```json
{
	"skip_public_range" : 0,
	"calendar": {
		"cal_id": "wcjgewCwAAqeJcPI1d8Pwbjt7nttzAAA",
		"admins":[
				"admin1",
				"admin2"
		],
		"summary": "test_summary",
		"color": "#FF3030",
		"description": "test_describe_1",
		"shares": [{
				"userid": "userid1",
				"permission": 1
			},
			{
				"userid": "userid2",
				"permission": 3
			}
		],
		"public_range": {
			"userids": ["abel", "jack"],
			"partyids": [1232, 34353]
		}
	}
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误信息 |
| fail_result | object | 无效的输入内容 |
| fail_result.shares | object[] | 无效的日历通知范围成员列表 |
| fail_result.shares.errcode | int32 | 错误码 |
| fail_result.shares.errmsg | string | 错误码说明 |
| fail_result.shares.userid | string | 日历通知范围成员的id |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg" : "ok",
	"fail_result": {
		"shares":[{
			"errcode": 40001,
			"errmsg": "not found",
			"userid": "userid3"
		}]
	}
}
```
