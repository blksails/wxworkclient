# 创建预约会议 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98148](https://developer.work.weixin.qq.com/document/path/98148)
- **文档 ID**: `98148`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/create?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于创建一个预约会议。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| admin_userid | string | 是 | 会议管理员userid |
| title | string | 是 | 会议的标题（1~255位字符长度） |
| meeting_start | uint32 | 否 | 会议开始时间的unix时间戳，需大于当前时间 |
| meeting_duration | uint32 | 否 | 会议持续时间（单位秒），最小300秒，最大86399秒 |
| description | string | 否 | 会议的描述，最多支持500个字节或者utf8字符 |
| location | string | 否 | 会议地点。最长支持18个汉字或36个英文字母。 |
| agentid | uint32 | 否 | 授权方安装的应用agentid。仅旧的第三方多应用套件需要填此参数 |
| invitees | object | 否 | 邀请参与会议的成员。人数上限以指定的「管理员」可预约的人数上限来校验：普通企业与会人员最多100付费企业由企业选购的在线会议室或高级账号对应的容量决定，但最多不能超过300人，若超过300人请调用更新会议受邀成员列表接口注意：任何userid不合法或者不在应用可见范围，直接报错。 |
| cal_id | string | 否 | 会议所属日历ID。该日历必须是access_token所对应应用所创建的日历。注意，若参与人在日历分享范围内，则插入到该日历（同时会插入会议参与人的默认日历），若不在分享范围内，否则仅插入到参与者默认日历；如果不填，那么插入到参与者的默认日历上。第三方应用必须指定cal_id不多于64字节 |
| area | string | 是 | 国家/地区代码（例如：中国传86，不是+86，也不是0086） |
| phone_number | string | 是 | 手机号 |
| guest_name | string | 否 | 会议嘉宾姓名（1~16位字符长度） |
| password | string | 否 | 入会密码，仅可输入4-6位纯数字 |
| enable_waiting_room | bool | 否 | 是否开启等候室。true：开启等候室false：不开启等候室默认不开 |
| allow_enter_before_host | bool | 否 | 是否允许成员在主持人进会前加入。true：允许false：不允许默认允许 |
| enable_enter_mute | uint32 | 否 | 成员入会时静音设置。1：开启0：关闭2：超过6人后自动开启静音默认超过6人自动开启静音 |
| allow_unmute_self | bool | 否 | 允许参会者取消静音。true：允许false：不允许默认允许 |
| mute_all | bool | 否 | 是否设置全体静音。该字段单独使用不生效，当设置allow_unmute_self为false时，需要指定mute_all=true |
| watermark_type | uint32 | 否 | 水印样式。当屏幕共享水印参数为开启时，此参数才生效。0：单排1：多排默认为单排 |
| auto_record_type | string | 否 | 自动会议录制类型。none：禁用，表示不开启自动会议录制local：本地录制，表示主持人入会后自动开启本地录制cloud：云录制，表示主持人入会后自动开启云录制。说明：该参数依赖企业账户设置，当企业强制锁定后，该参数必须与企业配置保持一致。 |
| attendee_join_auto_record | bool | 否 | 是否当有参会成员入会时立即开启云录制。false：关闭，主持人入会自动开启云录制true：开启，有参会成员入会自动开启云录制默认为关闭说明：该参数必须 auto_record_type 设置为“cloud”时才能配置，该参数依赖企业账户设置，当企业强制锁定后，该参数必须与企业配置保持一致。 |
| enable_host_pause_auto_record | bool | 否 | 是否允许主持人暂停或者停止云录制。true：开启，主持人允许暂停和停止云录制false：不允许，主持人不允许暂停和关闭云录制默认为允许说明：该参数必须 auto_record_type 设置为“cloud”时才能配置，该参数依赖企业账户设置，当企业强制锁定后，该参数必须与企业配置保持一致。 |
| enable_interpreter | bool | 否 | 同声传译开关。false：不开启true：开启同声传译默认为不开启 |
| enable_enroll | bool | 否 | 是否激活报名 |
| enable_host_key | bool | 否 | 是否开启主持人密钥。true：开启false：关闭默认为不开启 |
| host_key | string | 否 | 主持人密钥，仅支持6位数字。如开启主持人密钥后没有填写此项，将自动分配一个6位数字的密钥。 |
| allow_external_user | bool | 否 | 是否允许外部成员入会。true：所有成员可入会false：仅企业内部成员可入会默认所有成员可入会 |
| enable_screen_watermark | bool | 否 | 是否开启屏幕水印。true：开启false：不开启默认不开启 |
| hosts | object | 否 | 会议主持人列表，最多10个。若包含admin_userid，会自动过滤。任何userid不合法，直接报错。仅购买了高级功能的企业可以指定主持人。 |
| remind_scope | uint32 | 否 | 会议开始时来电提醒方式，1.不提醒 2.仅提醒主持人 3.提醒所有成员入 4.指定部分人响铃。默认仅提醒主持人 |
| ring_users | object | 否 | 指定响铃的成员列表。如果remind_scope为4，但是ring_users为空，则全部成员均不响铃。 |
| is_repeat | uint32 | 否 | 是否是周期性会议。1：周期性会议0：非周期性会议默认为0 |
| repeat_type | uint32 | 否 | 周期性会议类型，当is_repeat为1时有效。目前支持如下类型：0 - 每日1 - 每周2 - 每月7 - 工作日当指定为自定义重复时，仅能选择按每日、每周或者每月重复 |
| is_custom_repeat | uint32 | 否 | 是否自定义重复设置，当is_repeat=1时生效。0-否1-是 |
| repeat_interval | uint32 | 否 | 重复间隔当指定为自定义重复时必填该字段随repeat_type不同而含义不同例如：repeat_interval指定为3，repeat_type指定为每周重复，那么每3周重复一次；repeat_interval指定为3，repeat_type指定为每月重复，那么每3个月重复一次 |
| repeat_day_of_week | uint32[] | 否 | 每周周几重复仅当指定为自定义重复且重复类型为每周时有效取值范围：1 ~ 7，分别表示周一至周日。注意：自定义按周重复时，会议开始时间对应的那天需要包含在该数组中。例如：开始时间是周一，则该数组需要包含1，如果开发者没有指定，后台会补上 |
| repeat_day_of_month | uint32[] | 否 | 每月哪几天重复仅当指定为自定义重复且重复类型为每月时有效取值范围：1 ~ 31，分别表示1~31号。注意：自定义按月重复时，会议开始时间对应的那天需要包含在该数组中。例如：开始时间是某月1号，则该数组需要包含1，如果开发者没有指定，后台会补上 |
| repeat_until_type | uint32[] | 否 | 结束重复类型。0：按日期结束重复1：按次数结束重复默认为0 |
| repeat_until_count | uint32 | 否 | 周期会议限定次数。说明：每天、每个工作日、每周最大支持200场子会议每月最大支持50场子会议如未填写，则默认为7次 |
| repeat_until | uint32 | 否 | 周期会议结束时刻。repeat_until_type=0时该字段才生效。。每天\每个工作日\每周 最多重复200次会议；每月最多重复50次会议。如未填写，默认为当前日期往后推7天 |
| remind_before | uint32 | 否 | 指定会议开始前多久提醒成员，相对于meeting_start前的秒数。目前仅支持0：会议开始时提醒300：5分钟前提醒900：15分钟前提醒3600：一小时前提醒86400：一天前提醒。默认不提醒，若指定了非支持的值，则表现为会议开始时提醒 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| invitees.userid | string[] | 参与会议的企业成员userid |
| guests | object[] | 会议嘉宾列表，详见Guest |
| settings | object | 会议配置，详见Settings |
| reminders | object | 周期性相关配置，详见Reminders |
| hosts.userid | string[] | 企业成员userid |
| ring_users.userid | string[] | 指定响铃的成员userid |
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| meetingid | string | 会议id，通过此id可调用“进入会议”接口(包括小程序接口和JS-SDK接口)，以向参会人提供便捷入会入口 |
| excess_users | string | 参会人中包含无效会议账号的userid，仅在购买会议专业版企业由于部分参会人无有效会议账号时返回 |
| meeting_code | string | 会议的会议号 |
| meeting_link | string | 入会链接 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","meetingid":"XXXXXXXXX","excess_users":["zhangsan","lisi"],"meeting_code":"meeting_code","meeting_link":"https://meeting.com/jige"}
```
