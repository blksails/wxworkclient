# 扫描推广二维码事件

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98071](https://developer.work.weixin.qq.com/document/path/98071)
- **文档 ID**: `98071`
- **请求方法**: `POST`
- **接口地址**: `https://127.0.0.1/service/receive`

## 接口描述

与本服务商已经有第三方应用授权关系的企业，其成员扫描推广二维码时触发该事件，回调到服务商通用开发参数设置的系统事件接收URL上。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| msg_signature | string | 是 | 消息签名 |
| timestamp | string | 是 | 时间戳 |
| nonce | string | 是 | 随机数 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ServiceCorpId | string | 是 | 服务商corpid |
| InfoType | string | 是 | 事件类型，固定为enter_register_package |
| TimeStamp | string | 是 | 回调事件的时间戳 |
| AuthCorpId | string | 是 | 扫描推广二维码的成员所属企业corpid |
| AuthUserInfo.UserId | string | 是 | 成员的userid |
| RegisterCode | string | 是 | 注册链接对应的注册码 |
| State | string | 否 | 自定义的状态值，参数值由接口 获取注册码 指定。若未指定，则无该字段 |
| TemplateId | string | 是 | 推广包ID |

### 请求示例

```xml
<xml>
	<ServiceCorpId><![CDATA[wwddddccc7775555aab]]></ServiceCorpId>
	<InfoType><![CDATA[enter_register_package]]></InfoType>
	<TimeStamp>1671632600</TimeStamp>
	<AuthCorpId><![CDATA[wwddddccc7775555aaa]]></AuthCorpId>
	<AuthUserInfo>
		<UserId><![CDATA[zhangshan]]></UserId>
	</AuthUserInfo>
	<RegisterCode><![CDATA[pIKi3wRPNWCGF-YU5KWjD]]></RegisterCode>
	<State><![CDATA[state1]]></State>
	<TemplateId><![CDATA[tpl1test]]></TemplateId>
</xml>
```
