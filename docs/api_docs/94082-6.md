# 查询客户接替状态 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94082](https://developer.work.weixin.qq.com/document/path/94082)
- **文档 ID**: `94082`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/externalcontact/resigned/transfer_result?access_token=ACCESS_TOKEN`

## 接口描述

企业和第三方可通过此接口查询离职成员的客户分配情况。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| handover_userid |  | 是 | 原添加成员的userid |
| takeover_userid |  | 是 | 接替成员的userid |
| cursor |  | 否 | 分页查询的cursor，每个分页返回的数据不会超过1000条；不填或为空表示获取第一个分页 |

### 请求示例

```json
{"handover_userid":"zhangsan","takeover_userid":"lisi","cursor":"CURSOR"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| customer.external_userid |  | 转接客户的外部联系人userid |
| customer.status |  | 接替状态， 1-接替完毕 2-等待接替 3-客户拒绝 4-接替成员客户达到上限 |
| customer.takeover_time |  | 接替客户的时间，如果是等待接替状态，则为未来的自动接替时间 |
| next_cursor |  | 下个分页的起始cursor |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","customer":[{"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACCCC","status":1,"takeover_time":1588262400},{"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACBBBB","status":2,"takeover_time":1588482400},{"external_userid":"woAJ2GCAAAXtWyujaWJHDDGi0mACAAAA","status":3,"takeover_time":0}],"next_cursor":"NEXT_CURSOR"}
```
