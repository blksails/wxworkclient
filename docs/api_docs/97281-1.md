# 家校通讯录批量变更事件 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97281](https://developer.work.weixin.qq.com/document/path/97281)
- **文档 ID**: `97281`

## 接口描述

当学校在家校通讯录中，“成员变更”或者“部门变更”时，若同一个学校短时间内触发多个变更事件，为了提高回调效率，则会将这些变更合并为一个批量的事件回调此到第三方应用/套件的指令回调URL。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| SuiteId |  | 否 | 第三方应用ID |
| AuthCorpId |  | 否 | 授权企业的CorpID |
| InfoType |  | 否 | 固定为change_school_contact_batch |
| TimeStamp |  | 否 | 最终回调的时间戳 |
| ChangeList |  | 否 | 变更列表。目前最大支持合并1000条，后续可能会根据情况调整，以实际收到的数据为准。 |
| ChangeType |  | 否 | 此处固定为create_student |
| Id |  | 否 | 学生的家校通讯录userid或者家长的id或者家校通讯录部门id |
| NewId |  | 否 | 当ChangeType为update_student或者update_parent时，变更后的学生的家校通讯录userid或者家长的id |
| TimeStamp |  | 否 | 产生变更的时间戳 |
