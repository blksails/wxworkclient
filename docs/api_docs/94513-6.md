# 获取成员授权列表

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94513](https://developer.work.weixin.qq.com/document/path/94513)
- **文档 ID**: `94513`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/user/list_member_auth`

## 接口描述

当企业当前授权模式为成员授权时，可调用该接口获取成员授权列表。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 企业调用接口凭证 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| cursor | string | 否 | 上一次调用时返回的next_cursor，第一次拉取可以不填 |
| limit | int32 | 否 | 每次拉取的数据量，默认值和最大值都为1000 |

### 请求示例

```json
{
	"cursor": "NEXT_KEY",
	"limit": 20
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取，返回空字符串代表已经是最后一页 |
| member_auth_list | object[] | 成员授权列表 |
| member_auth_list[].open_userid | string | 当用户为某个企业内部成员时返回。全局唯一，对于同一个服务商不同的应用open_userid是相同的；同一用户，对于不同服务商open_userid是不同的 |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"next_cursor":"next_cursor",
	"member_auth_list":[
		{
			"open_userid":"xxx"
		}
	]
}
```
