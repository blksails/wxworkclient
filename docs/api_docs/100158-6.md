# 创建部门 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/100158](https://developer.work.weixin.qq.com/document/path/100158)
- **文档 ID**: `100158`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/school/department/create?access_token=ACCESS_TOKEN`

## 接口描述

请求方式：POST（HTTPS）请求地址：https://qyapi.weixin.qq.com/cgi-bin/school/department/create?access_token=ACCESS_TOKEN请求包体：

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| name |  | 否 | 部门名称。长度限制为1~32个字符，字符不能包括-:*?"<>/，*，当设置了入学年份和标准年级时，该参数将被忽略 |
| parentid |  | 是 | 父部门id，32位整型 |
| id |  | 否 | 部门id，32位整型，指定时必须大于1。若不填该参数，将自动生成id |
| type |  | 是 | 部门类型，32位整型，1表示班级，2表示年级，3表示学段，4表示校区 |
| standard_grade |  | 否 | 标准年级，32位整型，参数值含义详见标准年级对照表，仅当部门类型为年级（2）时生效 |
| register_year |  | 否 | 入学年份，32位整型，格式为YYYY，输入范围为1970～2100，仅当部门类型为年级（2）时生效，如果在创建标准年级时不填此参数，则由系统自动计算出入学年份 |
| order |  | 否 | 在父部门中的次序值。order值大的排序靠前。有效的值范围是[0, 2^32) |
| department_admins |  | 否 | 部门管理员列表 |
| 自建应用 | 自建应用 | 否 | 配置到「家校沟通- 读取和编辑家校通讯录的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 需拥有「家校沟通」可使用家校通讯录-家校通讯录编辑权限 |
| 第三方应用 | 第三方应用 | 否 | 需拥有「家校沟通」使用和编辑权限 |

### 请求示例

```json
{"name":"一年级","parentid":5,"id":2,"type":1,"register_year":2018,"standard_grade":1,"order":1,"department_admins":[{"userid":"zhangsan","type":4,"subject":"语文"},{"userid":"lisi","type":3,"subject":"数学"}]}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| department_admins.userid |  | 对应管理端的账号，企业内必须唯一。不区分大小写，长度为1~64个字节 |
| department_admins.type |  | 部门管理员类型， 1表示校区负责人，2表示年级负责人，3表示班主任，4表示任课老师，5表示学段负责人 |
| department_admins.subject |  | 教师的科目，仅班主任和任课老师可以设置，科目的最多15个字符，仅支持设置一个科目 |
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| id |  | 创建的部门id |

### 响应示例

```json
{"errcode":0,"errmsg":"created","id":2}
```
