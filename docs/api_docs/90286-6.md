# 批量更新发票状态 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/90286](https://developer.work.weixin.qq.com/document/path/90286)
- **文档 ID**: `90286`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/card/invoice/reimburse/updatestatusbatch?access_token=ACCESS_TOKEN`

## 接口描述

接口说明：发票平台可以通过该接口对某个成员的一批发票进行锁定、解锁和报销操作。注意，报销状态为不可逆状态，请开发者慎重调用。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证 |
| reimburse_status |  | 是 | 发票报销状态 INVOICE_REIMBURSE_INIT：发票初始状态，未锁定；INVOICE_REIMBURSE_LOCK：发票已锁定，无法重复提交报销;INVOICE_REIMBURSE_CLOSURE:发票已核销，从用户卡包中移除 |
| invoice_list |  | 是 | 发票列表，必须全部属于同一个openid |
| card_id |  | 是 | 发票卡券的card_id |
| encrypt_code |  | 是 | 发票卡券的加密code，和card_id共同构成一张发票卡券的唯一标识 |

### 请求示例

```json
{"openid":"OPENID","reimburse_status":"INVOICE_REIMBURSE_INIT","invoice_list":[{"card_id":"cardid_1","encrypt_code":"encrypt_code_1"},{"card_id":"cardid_2","encrypt_code":"encrypt_code_2"}]}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| openid |  | 用户openid，可用"userid与openid互换接口"获取 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok",}
```
