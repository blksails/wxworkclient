# 获取会议详情 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98149](https://developer.work.weixin.qq.com/document/path/98149)
- **文档 ID**: `98149`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/get_info?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于获取指定会议的详情内容。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 否 | 会议id。注意：meetingid和meeting_code必须填一个 |
| meeting_code | string | 否 | 入会码。注意：meetingid和meeting_code必须填一个 |
| sub_meetingid | string | 否 | 周期性会议子会议id |
| sub_meetingid | string | 否 | 子会议 ID。 |
| status | uint32 | 否 | 子会议状态。0：默认（存在）1：已删除 |
| start_time | uint32 | 否 | 子会议开始时间（UTC 秒） |
| end_time | uint32 | 否 | 子会议结束时间（UTC 秒） |
| title | string | 否 | 子会议标题 |
| repeat_id | string | 否 | 周期性会议分段ID，通过该字段可以关联子会议所属的分段 |
| repeat_id | string | 否 | 分段ID |
| repeat_type | uint32 | 否 | 重复类型。0 - 每日1 - 每周2 - 每月7 - 工作日 |
| is_custom_repeat | uint32 | 否 | 是否自定义重复设置。0-否1-是 |
| repeat_interval | uint32 | 否 | 周期间隔该字段随repeat_type不同而含义不同例如：repeat_interval指定为3，repeat_type指定为每周重复，那么每3周重复一次；repeat_interval指定为3，repeat_type指定为每月重复，那么每3个月重复一次 |
| repeat_day_of_week | uint32[] | 否 | 每周周几重复取值范围：1 ~ 7，分别表示周一至周日 |
| repeat_day_of_month | uint32[] | 否 | 每月哪几天重复取值范围：1 ~ 31，分别表示1~31号 |
| repeat_until_type | uint32 | 否 | 周期性会议结束类型。0：按日期结束重复1：按次数结束重复 |

### 请求示例

```json
{"meetingid":"XXXXXXXXX","meeting_code":"meetingcodexxx","sub_meetingid":"submeeting"}
```
