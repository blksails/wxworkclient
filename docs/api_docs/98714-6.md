# 获取成员会议ID列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98714](https://developer.work.weixin.qq.com/document/path/98714)
- **文档 ID**: `98714`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/get_user_meetingid?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于获取指定成员指定时间内的会议ID列表。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| userid | string | 是 | 企业成员的userid |
| cursor | string | 否 | 上一次调用时返回的cursor，初次调用可以填"0" |
| limit | uint32 | 否 | 每次拉取的数据量，默认值和最大值都为100 |
| begin_time | uint32 | 否 | 开始时间 |
| end_time | uint32 | 否 | 结束时间，时间跨度不超过180天。如果begin_time和end_time都没填的话，默认end_time为当前时间。注意：end_time仅用于约束历史会议，对于未开始的会议，只要开始时间大于等于begin_time都会返回 |

### 请求示例

```json
{"userid":"USERID","cursor":"cursor","begin_time":1586136317,"end_time":1586236317,"limit":100}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取，next_cursor为未返回或者为空字符串表示数据已取完 |
| meetingid_list | string[] | 会议ID列表，可能为空 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","next_cursor":"1223","meetingid_list":["meetingid1","meetingid2"]}
```
