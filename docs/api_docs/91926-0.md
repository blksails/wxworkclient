### 订单价格

订单价格的计算，由多个方面构成，可参考如下图：
首先，订单产生时会指定版本、购买人数及购买时长。根据服务商在平台对应用定价策略的配置，可计算得到版本价格。其次，在扩容/版本变更时，会折算当前生效版本的价值，作为折扣金额扣除，得到定价价格。若不需要折算的情况，定价价格等同于版本价格。再次，服务商可在定价价格基础了，减免一定的优惠金额，得到的价格再乘以一个应付系数 ，得到应付价格。（ 应付系数在企业下单时为“1.0”；若为应用服务商代下单，则“平台分成比例”；若为代理服务商代下单，则为“1.0 - 代理商约定分成比例” ）最后，用户按该应付价格进行支付，完成订单。
在API接口获取的订单，获取到的价格，就是对应到上述的应付价格。
订单完成后，产生的资金，将按平台规则进行分成。分成包括三部分，分别服务商，平台及代理商（仅当该企业应用被应用代理商报备时存在），在API接口也会相应提供给到开发者，以供数据对账。
