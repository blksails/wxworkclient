## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
|  | string[] | 否 | 蓝牙设备主 service 的 uuid 列表某些蓝牙设备会广播自己主 service 的 uuid。如传入列表非空则只会搜索发出广播包有这个主服务的蓝牙设备，建议主要通过该参数过滤掉周边
不需要处理的其他蓝牙设备 |
|  | boolean | 否 | 是否允许重复上报同一设备若允许重复上报，则 onDeviceFound 方法会多次上报同一设备，但是 RSSI 值会有不同 |
|  | number | 否 | 上报设备的间隔若为 0 表示找到新设备立刻上报，否则根据传入的间隔上报 |
|  | Function | 否 | 成功回调 |
|  | Function | 否 | 失败回调 |
|  | Function | 否 | 取消回调 |
|  | Function | 否 | 完成回调 |

### 请求示例

#### 示例 1: 功能描述

```json
ww.startBluetoothDevicesDiscovery({services:['FEE7']})
```

#### 示例 2: 示例代码

```json
// 以微信硬件平台的蓝牙智能灯为例，主服务的 UUID 是 FEE7。传入这个参数，只搜索主服务 UUID 为 FEE7 的设备wx.startBluetoothDevicesDiscovery({services:['FEE7'],success:function(res){console.log(res)}})
```

### ww.startBluetoothDevicesDiscovery([params])

兼容性：企业微信
功能描述开始搜寻附近的蓝牙外围设备。
ww.startBluetoothDevicesDiscovery({services:['FEE7']})
参数说明params: Object属性类型默认值必填说明servicesstring[]否蓝牙设备主 service 的 uuid 列表某些蓝牙设备会广播自己主 service 的 uuid。如传入列表非空则只会搜索发出广播包有这个主服务的蓝牙设备，建议主要通过该参数过滤掉周边
不需要处理的其他蓝牙设备allowDuplicatesKeyboolean否是否允许重复上报同一设备若允许重复上报，则 onDeviceFound 方法会多次上报同一设备，但是 RSSI 值会有不同intervalnumber0否上报设备的间隔若为 0 表示找到新设备立刻上报，否则根据传入的间隔上报successFunction否成功回调failFunction否失败回调cancelFunction否取消回调completeFunction否完成回调
返回说明Promise<Object>属性类型必填说明errMsgstring是通用错误信息errCodenumber是通用错误码

### 功能描述

开始搜寻附近的蓝牙外围设备。

### params: Object

属性类型默认值必填说明servicesstring[]否蓝牙设备主 service 的 uuid 列表某些蓝牙设备会广播自己主 service 的 uuid。如传入列表非空则只会搜索发出广播包有这个主服务的蓝牙设备，建议主要通过该参数过滤掉周边
不需要处理的其他蓝牙设备allowDuplicatesKeyboolean否是否允许重复上报同一设备若允许重复上报，则 onDeviceFound 方法会多次上报同一设备，但是 RSSI 值会有不同intervalnumber0否上报设备的间隔若为 0 表示找到新设备立刻上报，否则根据传入的间隔上报successFunction否成功回调failFunction否失败回调cancelFunction否取消回调completeFunction否完成回调

### 返回说明

Promise<Object>
属性类型必填说明errMsgstring是通用错误信息errCodenumber是通用错误码
