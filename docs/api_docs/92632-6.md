# 提交审批申请

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/92632](https://developer.work.weixin.qq.com/document/path/92632)
- **文档 ID**: `92632`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/oa/applyevent?access_token=ACCESS_TOKEN`

## 接口描述

服务商可通过第三方应用Secret调用本接口，代应用可见范围内员工在企业微信“审批应用”内提交由此应用添加的审批类型对应的审批申请。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。必须使用第三方应用accesstoken获取，获取方式参考：文档-获取access_token |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| creator_userid | string | 是 | 申请人userid，此审批申请将以此员工身份提交，申请人需在应用可见范围内 |
| template_id | string | 是 | 模板的唯一标识id。可在“获取审批单据详情”、“审批状态变化回调通知”中获得，也可在使用“复制/更新模板到企业”接口回调中获得。注：此id为企业内模板的实例id，非服务商后台对应模板的id。暂不支持通过接口提交打卡补卡调班模板审批单。 |
| use_template_approver | int32 | 是 | 审批人模式：0-通过接口指定审批人、抄送人（此时process参数必填）; 1-使用此模板在管理后台设置的审批流程(需要保证审批流程中没有“申请人自选”节点)，支持条件审批。默认为0 |
| choose_department | int32 | 否 | 提单者提单部门id，不填默认为主部门 |
| process | object | 是 | 新版流程列表 |
| apply_data | object | 是 | 审批申请数据，可定义审批申请中各个控件的值，其中必填项必须有值，选填项可为空，数据结构同“获取审批申请详情”接口返回值中同名参数“apply_data” |
| summary_list | object[] | 是 | 摘要信息，用于显示在审批通知卡片、审批列表的摘要信息，最多3行 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码，详情见错误码说明 |
| errmsg | string | 错误码对应的错误信息提示 |
| sp_no | string | 表单提交成功后，返回的表单编号 |

### 响应示例

```json
{
   "errcode": 0,
   "errmsg": "ok",
   "sp_no": "202001010001"
}
```
