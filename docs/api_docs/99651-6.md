# 获取会议发起记录 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99651](https://developer.work.weixin.qq.com/document/path/99651)
- **文档 ID**: `99651`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/statistics/get_start_list?access_token=ACCESS_TOKEN`

## 接口描述

该接口用于获取员工发起的会议记录及其状态，包含员工主动发起快速会议及作为首位参与者进入预约会议的场景。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| type | int | 是 | 查询类型。1:发起成功的会议记录2:发起失败的会议（企业同时发起的会议数已达上限，员工无法发起） |
| begin_time | int | 是 | 查询范围起始时间戳，单位为秒 |
| end_time | int | 是 | 查询范围结束时间戳，单位为秒 |
| limit | int | 否 | 每次拉取的数据量，默认值200，最大值1000 |
| cursor | string | 否 | 用于分页查询的游标，由上一次调用返回，首次调用可不填 |

### 请求示例

```json
{"type":1,"begin_time":1700000000,"end_time":1700001000,"limit":1000,"cursor":"CURSOR"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取 |
| has_more | bool | 是否还有数据待拉取 |
| meeting_list | object[] | 发起会议成功或失败的记录信息，取决于请求中的type参数 |
| meeting_list.userid | string | 会议发起者的userid。如果是企业外部成员作为首位参与者进入预约会议，则该字段为此预约会议创建者的userid |
| meeting_list.start_time | int | 会议发起时间 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","next_cursor":"abcd","meeting_list":[{"userid":"USERID1","start_time":1700000000},{"userid":"USERID2","start_time":1700000000}]}
```
