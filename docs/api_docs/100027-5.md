# 会话消息搜索

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/100027](https://developer.work.weixin.qq.com/document/path/100027)
- **文档 ID**: `100027`
- **请求方法**: `SDK调用`

## 接口描述

可调用该接口对企业会话的内容进行搜索

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| query_word | string | 是 | 搜索的关键词，要求大于等于2个UTF8字符 |
| chat_info | object | 否 | 指定要搜索的客户会话范围 |
| start_time | uint32 | 否 | 指定搜索会话范围的起始时间，Unix时间戳 |
| end_time | uint32 | 否 | 指定搜索会话范围的结束时间，Unix时间戳。若指定了end_time，也必须指定start_time，start_time必须小于end_time，且时间范围不超过31天整；若都不指定则默认时间范围为当前时刻往前31天内 |
| skip_stop_words | bool | 否 | 指定是否在搜索时从搜索关键词中去掉部分常见停用词，以优化搜索结果。默认为false |
| limit | uint32 | 否 | 用于分页查询，每次请求返回的数据上限。最大为100，不填默认为50 |
| cursor | string | 否 | 用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chat_info.chat_type | uint32 | 否 | 指定会话类型，若填写则从指定的会话类型中搜索。1：单聊；2：群聊 |
| chat_info.id_list | object[] | 否 | chat_type为单聊时有效，会话双方的id，该列表长度必须为2 |
| chat_info.chat_id | string | 否 | chat_type为群聊时有效，若填写则从指定的群聊会话id中搜索 |
| chat_info.msg_type_list | uint32[] | 否 | 指定搜索的消息类型，仅支持可搜索的消息类型。如果不填则默认搜索所有可搜的消息类型 |
| chat_info.sender | object | 否 | 指定消息的发送者 |
| chat_info.sender_list | object[] | 否 | 指定消息的发送者列表，会搜索发送者为其中之一的消息。最多传入100个IdInfo |

### 请求示例

```json
{
	"query_word": "欢迎",
	"chat_info": {
		"chat_type": 1,
		"id_list": [ {
				"open_userid": "OPEN_USERID"
				}, {
				"external_userid": "EXTERNALID1"
			}
		],
		"chat_id": "CHAT_ID",
		"msg_type_list":[1],
		"sender": {
			"open_userid": "OPEN_USERID"
		},
		"sender_list": [
			{
				"open_userid":"OPEN_USERID"
			}
		]
	},
	"start_time": 1668355200,
	"end_time": 1668528000,
	"skip_stop_words": true,
	"limit": 50,
	"cursor": "CURSOR"
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| has_more | int32 | 是否还有更多数据未获取，不填默认为0 0: 否 1: 是 |
| next_cursor | string | 下一次请求的cursor值 |
| msg_list | object[] | 搜索结果的消息列表，按消息发送时间降序排序 |
| msg_list[].msgid | string | 每条消息对应的msgid |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"has_more": 1,
	"next_cursor": "CURSOR",
	"msg_list":[
		{
			"msgid": "xxxmsgid"
		},
		{
			"msgid": "yyymsgid"
		}
	]
}
```
