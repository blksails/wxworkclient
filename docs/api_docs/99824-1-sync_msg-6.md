# 获取会话记录

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/99824](https://developer.work.weixin.qq.com/document/path/99824)
- **文档 ID**: `99824`
- **API 名称**: `sync_msg`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/chatdata/sync_msg`
- **分组信息**: 第 1 个接口，共 2 个

## 接口描述

通过接口，获取企业一段时间内的会话记录。

## 请求信息

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| cursor | string | 否 | 上一次调用时返回的next_cursor，第一次拉取可以不填。若不填，从5天内最早的消息开始返回。不多于128字节 |
| token | string | 否 | 回调事件返回的 `token` 字段，10分钟内有效；建议都从回调事件中取出token填上，否则接口会有严格的频率限制。不多于128字节。 |
| limit | uint32 | 否 | 拉取数量，默认值200，最大值1000 |
| mode | uint32 | 否 | 消息拉取模式。0：默认模式；1-预分页模式；2-自适应模式。详见拉取模式 |
| begin_time | string | 否 | 仅用于需要重新拉取数据而不清楚具体cursor的场景，正常情况不应该使用该字段。 若该字段有值，会从begin_time附近的数据开始返回。 |

### 请求示例

```json
{
	"cursor":"RMTJID",
	"token":"NGEJKGOEGJKOEGNEOAGOEGOKEGHOEEEEE",
	"limit":200,
	"mode": 0
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
| has_more | uint32 | 是否还有更多数据。0-否；1-是。 |
| next_cursor | string | 下次调用带上该值，则从当前的位置继续往后拉，以实现增量拉取。强烈建议对该字段入库保存，每次请求读取带上，请求结束后更新。避免因意外丢，导致必须从头开始拉取，引起消息延迟。 |
| msg_list | object[] | 消息列表，按消息发送时间升序排序 |
