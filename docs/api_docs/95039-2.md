# 产生会话回调事件 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/95039](https://developer.work.weixin.qq.com/document/path/95039)
- **文档 ID**: `95039`

## 接口描述

为了提升企业会话存档的使用性能，降低无效的轮询次数。当企业收到或发送新消息时，企业微信可以以事件的形式推送到企业指定的url。回调间隔为15秒，在15秒内若有消息则触发回调，若无消息则不会触发回调。接收并解析事件的方法见：接收事件

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ToUserName |  | 否 | 企业微信CorpID |
| FromUserName |  | 否 | 此事件该值固定为sys，表示该消息由系统生成 |
| CreateTime |  | 否 | 消息创建时间（整型） |
| MsgType |  | 否 | 消息类型，此时固定为：event |
| AgentID |  | 否 | 企业应用的id，整型。可在应用的设置页面查看 |
| Event |  | 否 | 事件类型，此时固定为：msgaudit_notify |

### 请求示例

```xml
<xml><ToUserName><![CDATA[CorpID]]></ToUserName><FromUserName><![CDATA[sys]]></FromUserName><CreateTime>1629101687</CreateTime><MsgType><![CDATA[event]]></MsgType><AgentID>2000004</AgentID><Event><![CDATA[msgaudit_notify]]></Event></xml>
```
