# 为打卡人员排班

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94209](https://developer.work.weixin.qq.com/document/path/94209)
- **文档 ID**: `94209`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/checkin/setcheckinschedulist`

## 接口描述

企业可通过具有调用权限的应用，为打卡规则为“按班次上下班”规则的指定员工排班。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方式参考：文档-获取access_token |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| items[].userid | string | 是 | 打卡人员userid |
| items[].day | int32 | 是 | 要设置的天日期，取值在1-31之间。联合yearmonth组成唯一日期 比如20201205 |
| items[].schedule_id | int32 | 是 | 对应groupid规则下的班次id，通过预先拉取规则信息获取，0代表休息 |
| groupid | int32 | 是 | 打卡规则的规则id，可通过“获取打卡规则”、“获取打卡数据”、“获取打卡人员排班信息”等相关接口获取 |
| yearmonth | int32 | 是 | 排班表月份，格式为年月，如202011 |

### 请求示例

```json
{
    "groupid": 226,
    "items": [
        {
            "userid": "james",
            "day": 5,
            "schedule_id": 234
        }
    ],
    "yearmonth": 202012
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 错误码描述 |

### 响应示例

```json
{
    "errcode": 0,
    "errmsg": "ok"
}
```

## 其他说明

### 权限说明

仅支持为打卡规则为“按班次上下班”
调用的应用需要满足如下的权限：
- 自建应用：配置到「打卡 - 可调用接口的应用」中
- 代开发应用：具有「打卡」权限
- 第三方应用：具有「打卡」权限

注：从2023年12月1日0点起，不再支持通过系统应用secret调用接口，存量企业暂不受影响 查看详情

### 调用频率

接口调用频率限制为60次/分钟。
