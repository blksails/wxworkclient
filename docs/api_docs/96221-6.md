# 修改门禁通行规则

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/96221](https://developer.work.weixin.qq.com/document/path/96221)
- **文档 ID**: `96221`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/devicedata/mod_accesscontrol_rule`

## 接口描述

第三方应用可修改企业已经存在的门禁规则。

## 请求信息

### Query 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 应用调用接口权限 |

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| rule_id | string | 是 | 企业微信返回的rule_id，门禁规则的标识id |
| device_sn_list | string | 是 | 设备SN列表 |
| rule_name | string | 是 | 门禁规则名称 |
| pass_rule | object | 否 | 门禁规则 |
| pass_rule.rule_list | string | 是 | rule为具体规则，其解析语法请参考附录[门禁放行规则语法说明] |
| pass_rule.effect_open_userid_list | string | 是 | 生效成员的openuserid列表 |
| open_userid | string | 是 | 生效成员的openuserid |
| user_type | uint32 | 否 | 成员类型：0：企业员工 2：学生 不填则默认值为0 |
| remote_pass_rule | object | 否 | 远程开门规则，远程开门规则列表，object内各字段含义与 pass_rule_list相同 |

### 请求示例

```json
{
    "rule_id":"xxx",
    "device_sn_list":["SN"],
    "rule_name":"access_rule_name_xxx",
    "pass_rule":{
        "rule_list":[
            "9:00-10:00 * * 1-5 *",
            "9:00-10:00,15:30-17:45 * * 0,6 *"
        ],
        "effect_open_userid_list":[
            {
                "open_userid":"user1",
                "user_type":0
            },
            {
                "open_userid":"user2",
                "user_type":2
            }
        ]
    },
    "remote_pass_rule":{
        "rule_list":[
            "9:00-10:00 * * 1-5 *",
            "9:00-10:00,15:30-17:45 * * 0,6 *"
        ],
        "effect_open_userid_list":[
            {
                "open_userid":"user1",
                "user_type":0
            },
            {
                "open_userid":"user2",
                "user_type":2
            }
        ]
    }
}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 错误码 |
| errmsg | string | 错误码描述 |
| invalid_list | string | 非法的open_userid列表，errcode不为0时此字段有效 |

### 响应示例

```json
{
	"errcode": 0,
	"errmsg": "ok",
	"invalid_list": []
}
```
