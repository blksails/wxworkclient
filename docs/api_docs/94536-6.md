# 添加事件类别 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/94536](https://developer.work.weixin.qq.com/document/path/94536)
- **文档 ID**: `94536`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/report/grid/add_cata?access_token=ACCESS_TOKEN`

## 接口描述

可以添加一级或二级事件类别，网格员在提交「巡查上报」或「居民上报」时将填入配置的事件类别

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token |  | 是 | 调用接口凭证。“获取access_token” |
| category_name |  | 是 | 分类名称，不能超过30个字,同一一级分类下的二级分类名字不能一样 |
| level |  | 是 | 分类层级，这里只能传1或者2 |
| parent_category_id |  | 否 | 所属的一级分类的id，level为2的话必传 |
| 自建应用 | 自建应用 | 否 | 配置到「居民联系 - 可调用接口的应用」中 |
| 代开发应用 | 代开发应用 | 否 | 支持，需要勾选「居民联系权限」 |
| 第三方应用 | 第三方应用 | 否 | 暂不支持 |

### 请求示例

```json
{"category_name":"category_name","level":2,"parent_category_id":"parent_category_id"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode |  | 返回码 |
| errmsg |  | 对返回码的文本描述内容 |
| category_id |  | 分类id |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","category_id":"category_id"}
```
