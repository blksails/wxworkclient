# 获取网络研讨会嘉宾列表 - 文档 - 企业微信开发者中心

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/98871](https://developer.work.weixin.qq.com/document/path/98871)
- **文档 ID**: `98871`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/meeting/webinar/list_guest?access_token=ACCESS_TOKEN`

## 接口描述

获取指定的网络研讨会嘉宾列表，支持通过会议 ID 或会议 Code 方式查询。

## 请求信息

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token | string | 是 | 调用接口凭证。获取方法查看“获取access_token” |
| meetingid | string | 否 | 网络研讨会ID。meetingid 和 meeting_code 二者必须送一个，二者都送时以 meeing_id 为准。 |
| meeting_code | string | 否 | 网络研讨会的会议号。meetingid 和 meeting_code 二者必须送一个，二者都送时以 meeing_id 为准。 |

### 请求示例

```json
{"meetingid":"MEETINGID","meeting_code":"CODE"}
```

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| guests | object | 网络研讨会嘉宾列表。详见GuestInfo。 |
| guest_type | uint32 | 嘉宾类型：1：内部嘉宾2：外部嘉宾3：链接嘉宾 |
| userid | string | 嘉宾的userid。 |
| area | string | 国家/地区代码，外部嘉宾必填（例如：中国传86，不是+86）。 |
| phone_number | string | 手机号，外部嘉宾必填。通过嘉宾链接接入会的嘉宾无此字段。 |
| guest_name | string | 嘉宾名称，外部嘉宾必填（1~16位字符长度）。 |
| email | string | 邮箱，外部嘉宾选填。通过嘉宾链接接入会的嘉宾无此字段。 |

### 响应示例

```json
{"errcode":0,"errmsg":"ok","guests":[{"guest_type":1,"userid":"USERID","area":"86","phone_number":"15900000001","guest_name":"NAME","email":""}]}
```
