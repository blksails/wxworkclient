# 发送会议邮件

## 基本信息

- **文档地址**: [https://developer.work.weixin.qq.com/document/path/97868](https://developer.work.weixin.qq.com/document/path/97868)
- **文档 ID**: `97868`
- **请求方法**: `POST`
- **接口地址**: `https://qyapi.weixin.qq.com/cgi-bin/exmail/app/compose_send?access_token=ACCESS_TOKEN`

## 接口描述

应用可以通过该接口发送会议邮件。

## 请求信息

### Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| to | object | 是 | 收件人，to.emails 和 to.userids 至少传一个 |
| to.emails | string[] | 否 | 收件人，邮箱地址 |
| to.userids | string[] | 否 | 收件人，企业内成员的userid |
| cc | object | 否 | 抄送 |
| cc.emails | string[] | 否 | 抄送人，邮箱地址 |
| cc.userids | string[] | 否 | 抄送人，企业内成员的userid |
| bcc | object | 否 | 密送 |
| bcc.emails | string[] | 否 | 密送人，邮箱地址 |
| bcc.userids | string[] | 否 | 密送人，企业内成员的userid |
| subject | string | 是 | 邮件标题，同时也是会议标题 |
| content | string | 是 | 邮件正文，同时是会议描述 |
| attachment_list | object[] | 否 | 附件相关 |
| attachment_list[].file_name | string | 是 | 文件名 |
| attachment_list[].content | string | 是 | 文件内容（base64编码），所有附件加正文的大小不允许超过50M, 且附件个数不能超过200个 |
| content_type | string | 否 | 内容类型 html，text（默认是html） |
| schedule | object | 是 | 会议相关数据，发会议邮件都必须带上这个字段 |
| schedule.schedule_id | string | 否 | 会议ID (修改/取消会议必须带上schedule_id) |
| schedule.method | string | 否 | 会议方法：  request-请求（不传schedule_id时是创建会议，传了是修改会议）   cancel-取消会议（必须带上schedule_id）   默认为request |
| schedule.location | string | 否 | 地点 |
| schedule.start_time | int32 | 是 | 会议开始时间，Unix时间戳 |
| schedule.end_time | int32 | 是 | 会议结束时间，Unix时间戳 |
| schedule.reminders | object | 否 | 重复和提醒相关字段 |
| schedule.reminders.is_remind | int32 | 否 | 是否有提醒 0-不提醒 1-提醒 |
| schedule.reminders.remind_before_event_mins | int32 | 否 | 会议开始（start_time）前多少分钟提醒，当is_remind=1时有效。 |
| schedule.reminders.timezone | int32 | 否 | 时区。UTC偏移量表示(即偏离零时区的小时数)，东区为正数，西区为负数。  例如：+8 表示北京时间东八区  默认为北京时间东八区   取值范围：-12 ~ +12 |
| schedule.reminders.is_repeat | int32 | 否 | 是否重复 0-否 1-是 |
| schedule.reminders.is_custom_repeat | int32 | 否 | 是否自定义重复 0-否 1-是。当is_repeat为1时有效。 |
| schedule.reminders.repeat_type | int32 | 否 | 重复类型，当is_repeat=1时有效。目前支持如下类型： 0 - 每日  1 - 每周  2 - 每月  5 - 每年 |
| schedule.reminders.repeat_interval | int32 | 否 | 重复间隔 仅当指定为自定义重复时有效，该字段随repeat_type不同而含义不同 |
| schedule.reminders.repeat_day_of_week | int32[] | 否 | 每周周几重复 仅当指定为自定义重复且重复类型为每周时有效  取值范围：1 ~ 7，分别表示周一至周日 |
| schedule.reminders.repeat_day_of_month | int32[] | 否 | 每月哪几天重复 仅当指定为自定义重复, 且重复类型为每月或每年时有效  取值范围：1 ~ 31，分别表示1~31号 |
| schedule.reminders.repeat_month_of_year | int32[] | 否 | 每年哪几个月重复 仅当指定为自定义重复且重复类型为每年时有效  取值范围：1 ~ 12，分别表示 1月 - 12月（每年重复需要repeat_month_of_year和repeat_day_of_month来指定某一天） |
| schedule.reminders.repeat_until | int32 | 否 | 重复结束时刻，Unix时间戳，当is_repeat=1时有效。不填或填0表示一直重复 |
| meeting | object | 否 | 会议相关，会议邮件必填，且必须同时带上schedule，会议的基本设置放在schedule里 |
| meeting.hosts | object | 否 | 会议主持人列表，最多10个。定义见收件人字段，只支持填userid |
| meeting.meeting_admins | object | 是 | 会议管理员字段, , 仅可指定1人，只支持传userid，必须是同企业的用户，且在参与人中 |
| meeting.option | object | 否 | 会议相关设置 |
| meeting.option.password | string | 否 | 入会密码，仅可输入4-6位纯数字 |
| meeting.option.auto_record | int32 | 否 | 是否自动录制  0：未开启自动录制，1：开启自动本地录制，2：开启自动云录制；默认不开启 |
| meeting.option.enable_waiting_room | bool | 否 | 是否开启等候室 false:不开启等候室；true:开启等候室；默认不开 |
| meeting.option.allow_enter_before_host | bool | 否 | 是否允许成员在主持人进会前加入。 true:允许；false:不允许。默认允许 |
| meeting.option.enter_restraint | int32 | 否 | 是否限制成员入会  0:所有人可入会 2:仅企业内部用户可入会；默认所有人可入会 |
| meeting.option.enable_screen_watermark | bool | 否 | 是否开启屏幕水印  true:开启；false:不开启。默认不开启 |
| meeting.option.enable_enter_mute | int32 | 否 | 成员入会时是否静音  1:开启；0:关闭；2:超过6人后自动开启静音。默认超过6人自动开启静音 |
| meeting.option.remind_scope | int32 | 否 | 会议开始时是否提醒  1:不提醒 2:仅提醒主持人 3:提醒所有成员入会; 默认仅提醒主持人 |
| meeting.option.water_mark_type | int32 | 否 | 水印类型  0:单排水印 1:多排水印；默认单排水印 |
| enable_id_trans | uint32 | 否 | 表示是否开启id转译，0表示否，1表示是，默认0。仅第三方应用需要用到，企业自建应用可以忽略。 目前仅subject、content、attachment_list[].file_name字段支持转译。 |

## 响应信息

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |

### 响应示例

```json
{
    "errcode": 0,
    "errmsg": "ok"
}
```
